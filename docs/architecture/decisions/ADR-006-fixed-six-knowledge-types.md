# ADR-006：固定六类知识类型 + 扩展注册

OpenClaw（sa-01 子Agent） | 2026-04-19

## 状态

Accepted

## 上下文

知识条目需要分类以支撑针对性的提取、检索和冲突检测策略。需要决定类型体系的开放程度。

## 决策

固定六类核心类型（fact / methodology / decision / experience / intent / meta），新类型通过注册机制扩展。注册时需提供类型定义、提取 prompt 模板和冲突检测规则。

## 后果

- 六类核心类型覆盖绝大多数知识分类场景，提供稳定的类型语义基础
- 冲突检测可针对不同类型定制判定逻辑（fact 的冲突判定与 decision 不同）
- 扩展注册机制保证可扩展性（QS-05），不因类型封闭限制未来需求
- 注册新类型需提供完整的三件套（定义 + prompt 模板 + 冲突规则），门槛适中
- 类型数量增长后需关注检索和冲突检测的性能影响

## 替代方案

| 方案 | 优势 | 劣势 | 否决理由 |
|------|------|------|----------|
| 自由标签（无固定类型） | 灵活度最高，用户自定义 | 缺乏结构约束，提取和冲突检测无法针对类型优化，标签膨胀后检索质量下降 | 冲突检测依赖类型语义（fact 的冲突判定与 decision 不同），自由标签无法支撑 |
| 固定类型不可扩展 | 实现简单，类型语义确定 | 无法适应未预见的知识分类需求 | 可扩展性是关键质量属性（QS-05），完全封闭不可接受 |
