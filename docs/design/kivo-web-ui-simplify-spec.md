# KIVO Web 控制台 UI 精简 — 产品需求规格

OpenClaw（pm-02 子Agent）| 2026-05-19

## 目标

将 KIVO Web 控制台从当前 10+ 导航入口的后台管理风格，精简为 5 个主入口的展示型工作台。精简后的页面应达到"可截图展示"级别——任何一屏截图都能让陌生人理解 KIVO 是什么、能做什么。

## 背景

当前 KIVO Web 控制台存在以下问题：
- 主导航 8 项 + 二级导航 2 项，功能堆叠像内部后台
- 多个页面信息密度无层级，用户不知道先看哪里
- 部分页面（审核、活动流）功能未完善或与其他页面重叠
- 截图缺少"主视觉"，无法一眼传达产品价值

## 约束

- C1: 只改前端代码（`web/` 目录），不改核心逻辑、API、数据库
- C2: 保持深色主题
- C3: 移动端适配（响应式布局）
- C4: 不删除任何 API 路由或后端逻辑，仅隐藏前端入口
- C5: 保留所有现有页面文件（不物理删除），通过导航和路由控制可见性

## 信息架构

### 主导航（5 项，始终可见）

| # | 入口 | 路由 | 图标 | 说明 |
|---|------|------|------|------|
| 1 | 仪表盘 | /dashboard | LayoutDashboard | 知识健康度 + 最近沉淀 + 覆盖度 + 命中率 |
| 2 | 知识库 | /knowledge | BookOpen | 列表 + 导入合并为一个入口 |
| 3 | 图谱 | /graph | Network | 知识图谱可视化（展示王牌页） |
| 4 | 搜索 | /search | Search | 语义搜索，一句话查询 |
| 5 | 冲突与裁决 | /conflicts | AlertTriangle | 知识冲突对照 + 一键裁决 |

### 隐藏/移除的入口

| 原入口 | 处理方式 | 原因 |
|--------|----------|------|
| 时间线 /timeline | 从主导航移除，总览页"最近变更"模块可跳转 | 与活动流重叠，信息量过大 |
| 调研 /research | 从主导航移除，仪表盘"薄弱领域"可跳转 | 运营功能，不适合展示 |
| 活动 /activity | 从主导航移除 | 与时间线重叠，展示价值低 |
| 导入 /knowledge/import | 合并到知识库页面内（Tab 或顶部按钮） | 减少独立入口 |
| 系统字典 /settings/dictionary | 移到设置齿轮图标下 | 管理员功能，不适合主导航 |
| 意图库 /settings/intents | 移到设置齿轮图标下 | 管理员功能 |
| 角色管理 /settings/roles | 移到设置齿轮图标下 | 管理员功能 |
| 审核 /review | 从导航移除 | 当前数据加载失败，未完善 |
| 分析子页 /analytics/* | 从导航移除，关键指标合并到仪表盘 | 多个子页分散注意力 |

### 设置齿轮（右上角或侧边栏底部）

点击展开下拉/抽屉，包含：系统字典、意图库、角色管理、规则分发状态。

## 功能需求

### FR-01: 导航精简

将 `app-shell.tsx` 中的 `primaryNavItems` 缩减为 5 项，`secondaryNavItems` 移除。被移除的页面路由仍可通过 URL 直接访问，但不在导航中展示。

**AC-01.1**: 主导航只显示 5 个入口：仪表盘、知识库、图谱、搜索、冲突与裁决  
**AC-01.2**: 侧边栏底部或顶栏右侧增加设置齿轮图标，点击展开管理员功能入口  
**AC-01.3**: 被隐藏的页面（时间线、调研、活动、审核、分析）通过 URL 仍可访问  
**AC-01.4**: 移动端汉堡菜单同步只显示 5 项主导航 + 设置齿轮

### FR-02: 仪表盘重构

将 `/dashboard` 首页从当前的"新手引导 + 统计卡片"改为"知识资产状态一览"。

**AC-02.1**: 已有数据时（知识条目 > 0），不再显示"第一次知识旅程"引导卡片  
**AC-02.2**: 首屏展示四个核心模块：
  - 知识资产概览（总数、本周新增、活跃/归档比例）
  - 最近被 Agent 命中的知识 Top 5（证明知识在被使用）
  - 待裁决冲突数 + 最新冲突摘要
  - Agent 问了但库里没答上的问题 Top 3（知识缺口信号）

**AC-02.3**: 每个模块点击可跳转到对应详情页  
**AC-02.4**: 信息层级三层结构：一句话结论 → 关键数字 → 可操作列表

### FR-03: 知识库页面精简

将 `/knowledge` 页面从功能堆叠改为聚焦核心操作。

**AC-03.1**: 首屏只保留：搜索框、类型筛选、状态筛选、"新建知识"按钮、"导入"按钮  
**AC-03.2**: 移除首屏常驻的：拖拽排序、表格/看板/密度切换、模板创建、导出、批量选择  
**AC-03.3**: 被移除的功能收纳到"更多操作"下拉菜单中  
**AC-03.4**: 列表默认只展示：标题、类型标签、状态、一句摘要、更新时间  
**AC-03.5**: "导入"按钮点击后展开导入面板或跳转到 `/knowledge/import`（合并入口）

### FR-04: 图谱页面优化

将 `/graph` 页面从调试工具风格改为展示王牌。

**AC-04.1**: 控制面板默认收起（折叠到左侧或底部小按钮）  
**AC-04.2**: 默认展示视角为"核心知识簇 + 冲突边 + 高频引用边"  
**AC-04.3**: 提供 3 个视角快捷按钮：主题视角、冲突视角、引用视角  
**AC-04.4**: 节点点击后右侧展示：知识摘要、来源、被哪些 Agent/任务引用过  
**AC-04.5**: 移除首屏的十几种关系/类型开关，收纳到展开的控制面板中

### FR-05: 搜索页面聚焦

将 `/search` 页面从筛选堆叠改为"一句话查询"体验。

**AC-05.1**: 首屏只保留一个大搜索框 + 3 个推荐问题示例  
**AC-05.2**: 类型、时间、置信度、标签等筛选折叠到"高级筛选"按钮下  
**AC-05.3**: 搜索结果每条显示：答案摘要、引用的知识条目、置信度、命中原因  
**AC-05.4**: "最近修改"列表从首屏主体移到搜索结果下方或侧边

### FR-06: 冲突裁决页面优化

将 `/conflicts` 页面从带新手引导的列表改为直接可操作的裁决工作台。

**AC-06.1**: 已有数据时删除"先走完第一次知识旅程"新手引导  
**AC-06.2**: 冲突卡片首行直接展示冲突内容摘要（如"端口 3001 是否允许 nohup 启动"），而非"冲突 #conflict-xxx"  
**AC-06.3**: A/B 两侧对照展示：来源、时间、可信度、影响范围  
**AC-06.4**: 裁决按钮（采纳 A / 采纳 B / 合并 / 双废弃）放在证据对照下方  
**AC-06.5**: 裁决原因输入框保留

### FR-07: 设置入口收纳

将分散的管理功能收纳到统一的设置入口。

**AC-07.1**: 侧边栏底部（退出按钮上方）增加齿轮图标  
**AC-07.2**: 点击齿轮展开设置面板或跳转设置页，包含：系统字典、意图库、角色管理  
**AC-07.3**: 设置面板使用与主内容区一致的深色主题

### FR-08: 登录页文案优化

**AC-08.1**: 产品标题下方增加一句定位文案："让 Agent 记住、检索、裁决和复用知识"  
**AC-08.2**: 保留用户名和密码输入  
**AC-08.3**: 右侧或背景展示知识图谱/召回链路的装饰性预览（静态图或简单动画）

---

## 非功能需求

- NFR-01: 所有页面切换响应时间 < 200ms（客户端路由）
- NFR-02: 移动端（< 768px）侧边栏自动收起为汉堡菜单
- NFR-03: 深色主题下所有文字对比度符合 WCAG AA（4.5:1）
- NFR-04: 导航精简后首屏加载的 JS bundle 不增加（移除的组件做 lazy import）

## 验收场景

| 场景 | 步骤 | 预期结果 |
|------|------|----------|
| S1: 导航数量 | 登录后查看侧边栏 | 只有 5 个主导航项 + 底部设置齿轮 |
| S2: 仪表盘无引导 | 有知识数据时访问 /dashboard | 不显示"第一次知识旅程"，显示四个核心模块 |
| S3: 知识库精简 | 访问 /knowledge | 首屏只有搜索、筛选、新建、导入 |
| S4: 图谱默认收起 | 访问 /graph | 控制面板默认折叠，图谱占满主区域 |
| S5: 搜索聚焦 | 访问 /search | 首屏只有大搜索框和推荐问题 |
| S6: 冲突无引导 | 有冲突数据时访问 /conflicts | 不显示新手引导，直接展示冲突列表 |
| S7: 隐藏页可访问 | 直接输入 /timeline URL | 页面正常加载，但导航中无高亮项 |
| S8: 移动端导航 | 768px 以下访问 | 汉堡菜单展开只有 5 项 + 设置 |
| S9: 设置入口 | 点击齿轮图标 | 展开包含字典、意图库、角色管理的面板 |

---

## 不做的事

- 不改 API 路由或后端逻辑
- 不改数据库 schema
- 不新增页面（只精简和重组现有页面）
- 不改变认证/授权机制
- 不做国际化（保持中文）
- 不改变图谱的底层渲染引擎
- 不做数据迁移
