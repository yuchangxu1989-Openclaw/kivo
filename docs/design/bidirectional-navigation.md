# 双向跳转规约

OpenClaw(ux-01 子Agent)｜2026-05-25

## 结论

关联知识必须能点过去，来源材料必须能打开全文，跳过去后必须能回来。

KIVO 的跳转是一条可逆链路：材料 → 知识 → Wiki → 图谱 → 相关知识 → 回到材料。

证据：FR-P01 AC9 要求从来源材料查看提取出的所有知识；FR-P06 AC2 要求 Wiki 页面自动维护双向链接；FR-P06 AC5 要求页面展示知识来源溯源；FR-P03 AC6 要求知识点详情页展示所有关联。

## 跳转对象

### 材料

材料是原始文件。点击后打开材料详情页，并定位到页码、段落、坐标或时间戳。

链接格式：`材料：《概率论.pdf》 · 第 42 页`

### 知识条目

知识条目是原子知识。点击后打开知识详情页。

链接格式：`知识：条件概率的定义`

### Wiki 页面

Wiki 页面是聚合产物。点击后打开 Wiki 页面，并保留领域上下文。

链接格式：`Wiki：条件概率`

### 图谱关系

图谱关系用于看一跳或多跳关联。点击“在图谱中查看”打开图谱并聚焦当前节点。

证据：当前 `space-manager.tsx` 里“关联知识”区域已有 `graphNodeHref || /graph?focus=<id>`；`related-entries.tsx` 中关联条目链接到 `/knowledge/<targetId>`。

## 点击行为

### 单击

单击直接打开目标，不再弹二次确认。

原因：用户问“点击关联知识能否去到对应的知识”，这里的预期是可达，不是先弹一个不确定层。

### 悬停

悬停显示轻量预览卡：标题、摘要、类型、来源、置信度。

预览卡只帮用户确认“我要不要点”，不承载完整阅读。

### 返回

每次跳转都保留来源上下文：

- 从 Wiki 点关联知识：知识页顶部显示“从 Wiki《条件概率》进入 · 返回”。
- 从知识点打开材料：材料页顶部显示“从知识《条件概率的定义》进入 · 返回”。
- 从材料点产出知识：知识页顶部显示“从材料《概率论.pdf》第 42 页进入 · 返回”。

返回优先使用应用内 breadcrumb，不依赖浏览器后退。

## 面包屑

领域上下文面包屑格式：

领域知识库 / 数学 / 概率论 / 条件概率 / 当前页面

材料上下文面包屑格式：

材料库 / 概率论.pdf / 第 42 页 / 当前知识

跨领域时面包屑保留进入路径，并在目标标题旁显示所属领域：

当前进入路径：概率论  
目标同时属于：统计推断、数据分析

证据：当前 `app-shell` 和 `sidebar-tree` 提供导航树；但实测 `web/components` 下没有稳定 breadcrumb 组件命中，Wiki 详情页顶部只显示 `{space} / Wiki`，还不是完整面包屑。

## Wiki → 来源材料

Wiki 页面“来源材料”区域每条来源都必须可点。

点击规则：

1. 如果来源是 PDF：同页打开右侧预览面板，定位到第 X 页并高亮段落。
2. 如果用户点“打开全文”：进入材料详情页，左侧目录保留，右侧 PDF 查看器定位到第 X 页。
3. 如果来源是图片：同页打开图片预览，自动框选识别区域；再点“查看原图”进入材料详情。
4. 如果来源是音频/视频：同页打开播放器，从时间戳开始播放。

证据：当前 `ContentRenderer` 的 sourceRefs 只渲染 `<a href={uri}>`，`target=_blank`；Wiki 聚合页“来源溯源”只展示标题和摘要，没有材料页定位行为。`pipeline-worker.ts` 写入 Wiki metadata 的 source URI 是 `material://<id>`，但页面还没有把它转成材料预览链接。

## 知识 → PDF / 图片全文

知识详情页的来源区域必须显示：

- 原材料文件名
- 页码/时间戳/坐标
- “查看原文”
- “下载原文件”
- “查看同材料产出的所有知识”

点击“查看原文”时，不打开抽取切片，而是打开原始 PDF 或图片全文，并定位到来源位置。

证据：当前 `knowledge/[id]/page.tsx` 有“来源”文本；`entries.source_json.reference` 可保存 `material://<id>`；但没有完整材料查看器和页码跳转。

## 关联知识发现路径

### Wiki → 关联 Wiki

Wiki 页面正文中的知识链接点击到另一个 Wiki 页面。底部“关联链接”按关系类型分组。

当前状态：`wiki_links` 表存在，但 DB 计数为 0。页面会提示“当前没有解析出的 wiki_links”。这说明 UI 位子有了，数据未贯通。

### Wiki → 来源材料

来源材料区列出所有原始文件，并按页码/时间戳定位。

当前状态：Wiki metadata 有 `source.uri=material://<id>`，页面展示未形成材料跳转。

### Wiki → 同领域其他知识

侧栏展示“同领域相关知识”：前置知识、相关题目、解题方法、用户批注、易混概念。

证据：FR-P03 AC6 明确这些分组。

### 知识 → 引用它的所有 Wiki

知识详情页展示“被哪些 Wiki 页面引用”。点击任意 Wiki 回到聚合页面。

当前状态：`backlinks-panel.tsx` 可展示反向链接到 `/knowledge/<sourceId>`，但 Wiki 引用关系还没统一。

### 材料 → 它产出的所有知识

材料详情页列出该材料产出的知识，按章节和条目类型分组。

证据：FR-P01 AC9 要求点击来源材料可查看该文件提取出的所有知识条目列表。

## 跨领域跳转

一个知识点可以属于多个领域目录节点。

跳转时保留进入上下文，不强行改用户所在领域。

示例：用户在“概率论”里点到“贝叶斯估计”，目标也挂在“统计推断”。页面显示：

- 当前位置：概率论 / 条件概率
- 该知识还属于：统计推断
- 切换到统计推断视角

证据：FR-P01 AC6 要求知识条目支持多标签，一个知识点可同时属于多个领域目录节点。

## 链接视觉规约

所有链接白底黑字，靠文字和边框区分，不使用暗色背景。

### 知识链接

样式：黑字 + 细下划线 + 小标签“知识”。

Hover：背景变浅灰，显示摘要预览。

### 材料链接

样式：黑色线框胶囊 + 文件图标 + 页码。

Hover：显示文件大小、处理状态、页码。

### Wiki 链接

样式：黑字 + “Wiki”标签 + 所属领域。

Hover：显示页面摘要和来源数。

### 外部链接

样式：黑字 + 外链箭头；点击新窗口打开。

材料、知识、Wiki 都在同窗口打开，避免用户迷路。只有外部网页新窗口。

## 当前实现差距

1. `wiki_links=0`，但 `graph_edges=6256`，关联关系双轨且数据不一致。
2. Wiki 页面有“来源溯源”和“关联链接”区块，但来源材料不能稳定打开到原 PDF 第 X 页。
3. `ContentRenderer` 能显示 sourceRefs，却只是普通外链，不认识 `material://`。
4. 知识页已有来源文字，但缺少“查看原文全文”的材料查看器入口。
5. 面包屑没有覆盖材料、知识、Wiki、图谱之间的返回路径。
6. 当前 entry 与 Wiki 的关联字段分散在 `source_json / metadata_json / wiki_links / graph_edges`，需要统一可导航关系模型。

## 验收口径

用户从 Wiki 页面点击“关联知识”后，应看到目标知识详情页，并能一键返回原 Wiki。

用户从知识详情页点击“查看原文”后，应打开原 PDF 全文第 X 页，不是只看到切片文本。

用户从材料详情页点击“产出知识”后，应看到该材料生成的所有知识，并能按章节、批注、题目、概念筛选。
