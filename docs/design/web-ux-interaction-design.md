# KIVO Web 页面交互设计

OpenClaw（ux-01 子Agent）| 2026-05-23

---

## 1. 设计总原则

KIVO 的 Web 入口要让第一次打开的人马上明白三件事：

- 这里现在有多少知识，健康不健康。
- 知识从哪里来，经过了什么处理，最后变成了哪些条目。
- 我能怎样搜索、浏览、追溯、采纳、纠错。

页面不展示演示数据，不展示写死指标。所有数字、列表、状态、日志都来自真实后端数据。拿不到数据时，页面直接说明“当前没有记录”或“数据加载失败”，并给出下一步，不用假数字撑场面。

全站采用浅色主题，导航固定，一级入口只有八个：仪表盘、意图知识库、领域知识库、知识搜索、知识图谱、调研队列、系统、设置。操作日志放到系统里，不再作为一级导航。

首次用户的默认路径是：

打开 KIVO → 看仪表盘知道系统里有什么 → 进入领域知识库上传/查看真实材料 → 看材料怎样变成知识 → 搜索一个问题 → 打开图谱看关联 → 查看调研队列里哪些报告能采纳入库。

## 2. 全局导航与页面骨架

### 2.1 侧边栏

侧边栏从上到下固定为：

1. 仪表盘
2. 意图知识库
3. 领域知识库
4. 知识搜索
5. 知识图谱
6. 调研队列
7. 系统
8. 设置

导航不再按“总览 / 知识库 / 图谱 / 调研”分组。第一次来的用户不需要理解内部分类，只需要看到完整产品能力。

“系统”是可展开入口，里面放：

- 操作日志
- 运行状态
- 导入记录
- 治理摘要

“设置”是可展开入口，里面放：

- 账号与密码
- 系统词典
- 权限范围
- 导入导出

### 2.2 顶部区

每个页面顶部保持同一结构：

- 左侧：页面标题 + 一句话说明这个页面能干什么。
- 右侧：当前页最关键的动作按钮。
- 标题下方：只放对用户有用的状态提醒，不放内部实现术语。

### 2.3 空状态

每个页面空状态都要说人话：

- 仪表盘空：还没有知识，先去导入材料或录入知识。
- 意图知识库空：还没捕捉到你的偏好和纠偏，系统会在对话后自动沉淀。
- 领域知识库空：先创建一个领域，再导入材料。
- 搜索空：换个问法，或先去导入材料。
- 图谱空：知识条目入库后才会出现图谱。
- 调研队列空：你在对话里发起调研后，这里会出现记录。
- 系统日志空：还没有系统事件。

## 3. 仪表盘 Dashboard

### 页面目标

仪表盘回答“我的知识系统现在怎么样”。用户不用点进每个页面，也能知道知识量、健康度、最近发生了什么、哪里需要处理。

### 第一次打开看到什么

用户首先看到四张核心卡片：

- 知识总量：当前 active 条目数，分意图知识和领域知识。
- 本周新增：本周新增条目数，以及较上周变化。
- 检索命中率：真实检索记录统计出来的命中率。
- 知识健康度：冲突、孤立节点、待补索引、失败导入的综合状态。

下面是三块内容：

- 最近活动：最近 10 条真实事件，例如“导入了 21 个文件”“概率论讲义产出 38 条知识”“调研报告被采纳”。
- 知识分布：按知识类型、领域、来源展示分布。
- 下一步建议：根据真实状态给一个最该处理的动作，例如“3 个文件提取失败，去材料库重试”。

### 核心操作流

用户打开仪表盘后：

1. 看顶部健康度，知道系统是否正常。
2. 看最近活动，确认系统最近有没有真正跑起来。
3. 点问题卡片进入对应页面：失败导入进领域知识库，低命中率进知识搜索，孤立节点进图谱。
4. 回到仪表盘确认处理后状态变化。

### 信息层级

第一层是健康结论，不让用户自己算：

- 健康：最近有新增、有检索命中、没有失败任务。
- 需关注：存在失败导入、待补索引、孤立知识或调研失败。
- 异常：关键数据源不可用、图谱无法加载、检索无向量结果。

第二层是指标：数量、趋势、分布。

第三层是明细入口：最近活动、问题列表、跳转按钮。

### 关键交互说明

- 所有指标都能点击下钻。例如“失败导入 2 个”点击后直接进入领域知识库材料列表，并自动筛选失败文件。
- 健康度卡片不显示内部英文状态值，统一显示中文状态。
- 最近活动可以按“知识变更 / 导入 / 调研 / 系统”快速筛选。
- 指标加载失败时展示具体哪块失败，不让整页空白。
- 仪表盘不能硬编码命中率、趋势或样例活动。

## 4. 意图知识库 Intent Knowledge

### 页面目标

意图知识库回答“系统理解了我哪些偏好、纠偏、规则和做事方式”。它服务 Agent 的意图理解，不和领域知识混在一起。

### 第一次打开看到什么

用户看到一个紧凑表格，表格上方有筛选条：

- 类型：偏好、纠偏、规则、决策、方法论、元认知。
- 功能：路由、质量门禁、上下文增强、决策支持、纠错。
- 来源：对话、文档、手动录入。
- 时间范围。

表格每行显示：

- 短标题
- 知识正文预览
- 类型标签
- 功能标签
- 所属域
- 来源
- 更新时间

### 核心操作流

用户想确认“系统有没有记住我说过的话”：

1. 在搜索框输入自然语言，比如“不要一小步一汇报”。
2. 系统返回语义相关的意图知识，按相关度排序。
3. 用户点开条目详情。
4. 详情页展示正文、正例、负例、相似说法、来源引用、版本历史、关联条目。
5. 用户可以编辑知识点、删除过时知识、还原历史版本。

### 信息层级

列表页只展示判断所需字段，不把详情塞满：

- 第一优先级：标题、正文预览、类型、功能。
- 第二优先级：来源、更新时间、所属域。
- 第三优先级：相似句、版本、关联关系，放到详情页。

详情页分为：

- 这条知识在说什么。
- 用户想要什么、不要什么。
- 它从哪里来。
- 它会影响哪些场景。
- 它和哪些知识相关。
- 它的历史版本。

### 关键交互说明

- 列表默认表格，不用卡片堆叠。
- 搜索必须走语义搜索，页面文案不要暗示关键词匹配。
- 条目只有 active 状态，不出现归档、废弃、待审核等状态筛选。
- 删除要二次确认，提示删除后不再参与注入和检索。
- 编辑提交后给出“已更新，正在重新检查冲突和关联”的反馈。
- 详情页不展示置信度字段。

## 5. 领域知识库 LLM Wiki / Domain Knowledge

### 页面目标

领域知识库回答“我的真实材料怎样变成可浏览、可搜索、可注入的领域知识”。这是 KIVO 的主工作台，必须能承载 20+ 个真实文件同时导入、处理、追溯和管理。

### 第一次打开看到什么

页面分成四个区：

- 左侧：领域空间列表，例如“概率论”“线性代数”“产品设计”。
- 中间左：当前领域的目录树。
- 中间右：知识条目列表或 Wiki 页面列表。
- 右侧：选中条目的详情、来源链路和图谱入口。

顶部有两个浏览模式切换：

- 目录树模式：适合按教材章节浏览。
- 标签云模式：适合跨目录找主题，例如“贝叶斯”“易错点”“公式推导”“考试高频”。

页面顶部还要有“材料库”入口，显示真实材料状态：总文件数、处理中、失败、已产出知识数。

### 核心操作流：创建领域

1. 用户点击“新建领域”。
2. 填名称、描述、知识范围。
3. 创建后出现空目录树和材料导入入口。
4. 用户可以新建目录，例如“概率论 → 条件概率 → 贝叶斯定理”。
5. 后续材料和条目都归入这个领域。

### 核心操作流：导入 20+ 个真实文件

用户导入 21 个学习材料时，页面要像一个真正的材料管理台：

1. 用户进入“材料库”。
2. 批量拖拽 PDF、图片、音频、视频、手写笔记。
3. 上传区展示文件总数、总大小、格式分布。
4. 用户选择目标领域和默认目录。
5. 点击开始导入后，文件进入材料列表。
6. 每个文件显示当前状态：等待处理、解析中、提取中、已提取、提取失败。
7. 处理完成后，每个文件显示产出：知识条目数、Wiki 页面数、图谱节点数。
8. 用户点开文件，看到它产出的所有知识条目和来源定位。

### 材料库列表设计

20+ 文件不能用大卡片铺满屏幕。材料库默认用紧凑表格：

每行显示：

- 复选框
- 文件名
- 文件类型图标：PDF、图片、音频、视频、手写笔记、Markdown、文本
- 所属领域
- 当前阶段
- 处理进度
- 产出数量
- 上传时间
- 操作

行内状态要清楚：

- 等待处理：灰色标签，说明还没开始。
- 解析中：进度条显示已解析页数、时长或帧数。
- 提取中：显示已处理分段数。
- 已提取：显示产出条目数。
- 提取失败：红色标签，显示失败原因入口。

列表顶部提供：

- 格式筛选：PDF、图片、音频、视频、文本。
- 状态筛选：全部、处理中、已提取、失败。
- 领域筛选。
- 搜索文件名。
- 排序：上传时间、文件大小、产出数量、状态。

### 批量操作

材料库支持批量选择：

- 全选当前页。
- 批量重新处理。
- 批量删除。
- 批量移动到某个领域。
- 批量打标签。
- 批量导出处理摘要。

批量删除必须二次确认，并说明会影响哪些产物。批量重处理要提示会重新生成知识条目和 Wiki 页面，已有条目如何处理由系统规则决定，页面只讲用户能理解的影响。

### 从文件到知识的转化可视化

每个材料详情页展示一条清晰链路：

原始文件 → 解析结果 → 分段/帧/页 → 候选知识 → 入库知识条目 → Wiki 页面 → 图谱节点 → 注入记录

每一步显示：

- 当前状态。
- 产出数量。
- 失败原因。
- 可点击明细。

例子：用户点开“浙大·概率论与数理统计.pdf”：

- 原始文件：PDF，612 页。
- 解析结果：已解析 612 页。
- 分段：128 个分段。
- 候选知识：420 条。
- 入库知识：137 条。
- Wiki 页面：32 页。
- 图谱节点：137 个。
- 注入记录：最近 7 天被引用 18 次。

这条链路是领域知识库的核心画面。用户要能一眼看到“一堆文件”已经被加工成了什么。

### 目录树模式

目录树模式适合教材、课程、研究资料：

- 左侧树形目录支持三到四层，不要无限嵌套。
- 目录节点显示该目录下条目数。
- 叶子和中间目录都能挂知识条目。
- 拖拽可以移动目录和条目。
- 目录右键或更多菜单支持新建子目录、重命名、删除。

选中目录后，右侧列表显示该目录下的知识条目。列表支持语义搜索、类型筛选、标签筛选、分页。

### 标签云模式

标签云模式适合跨章节浏览：

- 标签按出现频次和最近使用排序。
- 标签大小体现条目数量，颜色体现知识类型或领域。
- 点击一个标签后，右侧显示该标签下全部条目。
- 支持多标签组合筛选，例如“概率论 + 易错点 + 公式”。
- 标签下要显示来源分布，帮助用户知道这些知识主要来自哪些文件。

多标签是领域知识库的关键能力。一个知识点可以同时挂在目录“概率论 / 条件概率”和标签“贝叶斯”“公式推导”“考试高频”。

### 知识条目详情

详情页展示：

- 标题
- 类型标签
- 多标签
- 所属目录路径
- 正文
- 来源引用：文件名、页码、段落、时间戳或截图坐标
- 关联条目
- 所属 Wiki 页面
- 图谱入口
- 版本历史
- 最近注入记录

来源引用要能点击。点击后打开原始材料定位视图：PDF 定位到页码，音视频定位到时间戳，图片定位到区域，文本定位到段落。

### Wiki 页面展示

Wiki 页面是编译产物，用户浏览的是结构化页面，底层仍是原子知识条目。

页面结构建议：

- 概念解释
- 公式 / 定理
- 例题
- 易错点
- 关联概念
- 来源材料
- 最近更新

每个板块都能展开“由哪些原子知识组成”，用户可以追溯到具体材料。

### 关键交互说明

- 领域知识搜索必须走语义检索，不做关键词假搜索。
- 文件处理失败时，失败原因和重试入口放在同一行。
- 处理中的文件自动刷新，但刷新频率要温和，不让页面跳动。
- 已产出的知识条目可以从文件详情、目录树、标签云、搜索结果、图谱五个入口抵达。
- 用户手动创建、编辑、删除条目是补充入口，主路径仍是材料和调研报告自动加工。
- 页面不能把“空间”“页面”“条目”“材料”混成同一种卡片，要明确层级。

## 6. 知识搜索 Search

### 页面目标

知识搜索回答“我问一个自然语言问题，KIVO 能不能找到真正相关的知识”。它跨意图知识和领域知识检索。

### 第一次打开看到什么

页面顶部是大搜索框，placeholder 用自然语言：

“问一个问题，例如：我之前说过哪些调度规则？”

搜索框下方显示：

- 最近搜索。
- 推荐问题。
- 快速筛选：知识类型、领域、来源、时间。

### 核心操作流

1. 用户输入自然语言问题。
2. 按回车搜索。
3. 结果按语义相关度排序。
4. 每条结果展示标题、摘要、相关度、类型、领域、来源。
5. 用户点击结果进入条目详情。
6. 结果侧边展示“相关条目”和“图谱邻居”。

### 信息层级

结果页分三层：

- 主结果：最相关的知识条目。
- 相关扩展：图谱强关联条目。
- 搜索解释：为什么这条结果相关，用简短中文说明，不暴露算法名。

### 关键交互说明

- 搜索过程要有明显 loading，超过 1 秒显示“正在语义检索”。
- 无结果时给出三个建议：换问法、放宽筛选、去导入知识。
- 高级筛选默认收起。
- 键盘上下键可选择结果，回车打开。
- 搜索历史可删除。
- 页面不使用关键词高亮冒充语义理解；如果要高亮，只能作为阅读辅助，不作为相关性解释。

## 7. 知识图谱 Graph

### 页面目标

知识图谱回答“这些知识之间怎样连接，哪里是孤岛，哪里值得继续研究”。它是探索入口，不是纯展示海报。

### 第一次打开看到什么

用户看到图谱全局视图：

- 节点按知识类型着色。
- 边按关系类型区分。
- 左侧是筛选：领域、知识类型、时间范围、关系类型。
- 右侧是选中节点详情。
- 顶部是视图切换：全局、领域子图、冲突视图、来源视图、时间回放。

### 核心操作流

1. 用户打开图谱，默认看到全局。
2. 点击一个节点，周边节点高亮。
3. 右侧展示节点摘要、来源、关联条目、跳转详情。
4. 用户点击“只看这个领域”，图谱收敛到领域子图。
5. 用户点击“返回全局视图”，恢复全局。

### 信息层级

图谱区只负责看结构：

- 节点和边。
- 当前筛选条件。
- 选中节点。
- 洞察标记：孤立节点、桥接节点、稀疏社区、意外关联。

详情区负责讲清楚：

- 这个节点是什么。
- 它从哪里来。
- 它连接了谁。
- 它为什么重要。
- 下一步能做什么。

### 关键交互说明

- 图谱必须有“返回全局视图”。
- 连续点击节点时只保留最后一次选择，避免加载失败。
- 加载失败展示重试按钮和可读错误。
- 节点详情里提供“打开知识详情”“查看来源材料”“搜索相似知识”。
- 图谱洞察可以引导用户发起调研，但调研创建仍以对话为主；页面可以提示“复制这个方向去对话里发起调研”。
- 500 节点以内保持顺滑，节点多时自动聚合显示，不让页面卡死。


## 7.1 图谱模式视图

图谱模式视图让用户在同一张知识图谱里切换不同建模方式。入口常驻在图谱页，不跳页，不进入设置，不把模式选择藏到二级菜单。

交互原型：<http://43.163.80.95/prototypes/20260523-1641-kivo-graph-domain-domain-loading-skeleto.html>

### 模式切换入口

模式切换器放在图谱页顶部工具栏左侧，紧跟页面标题“知识图谱”之后。它是一个横向分段按钮组，桌面端展示为：

- 全部
- 通用
- 已注册模式 1
- 已注册模式 2
- 已注册模式 3

按钮组右侧保留一个“更多模式”下拉，只在已注册模式超过 5 个时出现。下拉里按模式名称排序，每项显示：模式图标、模式名称、节点类型数量、关系类型数量。用户点击任一项后，下拉收起，图谱原地重绘。

移动端和窄屏下，模式切换器收敛成一个下拉按钮，文案为当前模式名称，例如“全部模式”。点击后从底部弹出选择面板，面板顶部写“切换图谱模式”，下面依次列出“全部”“通用”和已注册模式。

默认规则：

- 没有注册任何模式时，默认展示通用图谱。
- 只注册 1 个模式时，默认仍展示全部，方便用户看到通用节点和模式节点的关系。
- 注册多个模式时，首次进入默认展示全部。
- 用户本次会话内切换过模式后，刷新页面保持上次选择。

切换动效：

1. 用户点击模式按钮。
2. 被点击按钮在 120ms 内切换为高亮状态。
3. 图谱画布覆盖一层半透明白色遮罩，中心显示“正在切换到：学术模式”。
4. 原有节点在 160ms 内淡出到 40% 透明度。
5. 新模式节点和边在 240ms 内从中心轻微扩散进入。
6. 右侧详情面板如果选中的节点仍在当前子图中，保留选中；如果不在，显示“当前模式未包含刚才选中的节点”。

切换不改变浏览器路由。页面 URL 不跳转，侧边栏不刷新，滚动位置保持。图谱只请求当前模式所需的节点、边、图例和聚合统计。

### 模式筛选与工具栏结构

图谱页顶部工具栏从左到右为：

1. 页面标题：知识图谱。
2. 模式切换器。
3. 搜索框：搜索当前模式内的节点。
4. “返回全局视图”按钮：只在用户进入领域子图、节点邻域、冲突视图时显示。
5. “新建知识”按钮。
6. 布局切换按钮：力导向、层级、社区。
7. 图例按钮。

左侧筛选栏在模式切换后同步更新。通用模式只展示领域、知识类型、时间范围、关系类型。学术模式额外展示“学科 domain”。其他注册模式展示该模式声明的专属字段筛选。筛选栏顶部固定显示当前模式名和本模式命中数量，例如“学术模式 · 126 个节点”。

### 图谱渲染差异

图谱画布必须让用户不用打开详情也能看出节点类型和关系类型。

节点样式：

- 通用节点：灰蓝圆点，使用文档图标。
- 概念：蓝色圆形，使用灯泡图标。
- 定理：紫色六边形，使用徽章图标。
- 公式：橙色圆角矩形，使用函数图标。
- 例题：绿色卡片形，使用练习本图标。
- 证明方法：红色菱形，使用路径图标。
- 应用场景：青色胶囊形，使用坐标图标。
- 未归类节点：浅灰圆点，显示问号角标。

边样式：

- 支撑 / 推导：蓝色实线箭头。
- 包含 / 组成：灰色实线，无箭头。
- 相似 / 相关：青色虚线。
- 冲突 / 不一致：红色虚线，边中点显示感叹号。
- 例证 / 应用：绿色点线箭头。
- 未归类关系：浅灰细线。

图例常驻在画布右下角，默认折叠成“图例”按钮。展开后显示当前模式实际用到的节点类型和关系类型，不展示当前模式没有出现的类型。用户悬停图例项时，画布中对应类型高亮，其他元素降到 25% 透明度；移开后恢复。

节点聚合遵循当前模式。全部视图下，聚合气泡按模式分组；单一模式下，聚合气泡按节点类型分组；学术模式下，优先按 domain 分组，再按节点类型分层。

### 节点类型选择器

“新建知识”按钮固定在图谱页顶部工具栏右侧。点击后从右侧打开抽屉，宽度 520px，移动端全屏。抽屉标题是“新建知识条目”。

抽屉字段顺序：

1. 知识内容。
2. 来源说明。
3. 节点类型。
4. domain / 领域标签。
5. 相关节点。
6. 保存按钮。

节点类型选择器放在“知识内容”下方，是必看但非必填字段。默认值为“通用节点”。选择器结构：

- 顶部显示当前选择：图标 + 节点类型名 + 所属模式。
- 点击后展开下拉面板。
- 下拉第一项固定为“通用节点”。
- 下面按模式分组展示已注册节点类型。
- 每个节点类型显示图标、颜色点、类型名称、字段提示，例如“公式 · 需要 LaTeX 或自然语言公式”。
- 支持输入过滤节点类型，过滤只按语义名称和显示名匹配，不改变已注册模式。

用户不选择时保存为通用节点。用户选择某个节点类型后，抽屉下方出现该类型的补充字段；必填字段用红色星号标记。切换节点类型时，如果已填写字段会被保留在“临时字段”里，并提示“切换类型后，未匹配字段不会丢失，但不会作为结构化字段保存”。

保存时的反馈：

- 保存按钮进入 loading 状态，文案变成“正在入库”。
- 成功后抽屉关闭，新节点以 400ms 的描边闪烁出现在当前图谱里。
- 如果当前模式不包含新节点，页面顶部提示“条目已保存到通用节点，当前模式未显示。可切换到全部查看”。
- 失败时抽屉不关闭，错误显示在对应字段下方。

### 学科知识视图

学术模式是一套通用节点类型，不为每个学科单独注册模式。学科通过 domain 标签区分。

当用户切换到“学术模式”时，左侧筛选栏顶部出现“学科”区块：

- 全部学科
- 概率论
- 线性代数
- 机器学习
- 统计学
- 更多学科

每个学科项右侧显示节点数量。点击某个学科后，图谱就地收敛到该 domain 子图，顶部工具栏显示筛选胶囊，例如“domain：概率论”。胶囊右侧有关闭按钮，点击后回到全部学科。

学科分组展示：

- 画布左上角显示当前学科摘要：节点数、关系数、孤立节点数、最近更新。
- 全部学科视图中，图谱按 domain 形成浅色分区背景，每个分区左上角显示学科名。
- 跨学科关系用更粗的浅紫色边展示，边上标记“跨学科”。
- 同一知识点被多个 domain 命中时，节点外圈显示双层描边，详情面板展示所有 domain。

新学科首次出现时，不要求用户预先创建。图谱页收到后端返回的新 domain 后，自动出现在学科筛选里，并在第一次出现时给出轻提示：“发现新学科：概率论，已加入筛选”。提示 4 秒后自动消失，不打断用户操作。

### 空状态

未注册任何模式时，图谱页不显示空白页，也不提示用户必须配置模式。它保持当前默认图谱能力：自由关系提取、统一节点样式、按领域和知识类型筛选。

空状态展示在顶部工具栏下方，是一条浅蓝提示条：

“当前还没有注册图谱模式。KIVO 会按默认方式展示知识关系；注册模式后，这里会出现模式切换和类型图例。”

提示条右侧有两个按钮：

- 查看默认图谱
- 了解模式配置

“查看默认图谱”只是关闭提示条，不改变图谱。“了解模式配置”打开设置页对应说明或外部文档入口，但不在图谱页弹复杂配置表单。

此时工具栏只显示“通用”状态徽标，不显示不可点击的模式按钮。新建知识抽屉里的节点类型选择器隐藏，只在保存时按通用节点入库。

### 错误与降级

模式列表加载失败时，图谱继续展示通用视图。工具栏模式位置显示“模式加载失败”，旁边提供“重试”。点击重试只重新请求模式列表，不刷新整页。

某个模式切换失败时，用户仍停留在原模式。顶部出现错误提示：“学术模式暂时无法加载，已保留当前视图。”提示里有“重试”和“查看默认图谱”。

注册模式里某个节点类型缺少图标或颜色时，使用默认图标和默认颜色，右下角图例标记“使用默认样式”。这个问题记录到系统日志，不阻断用户看图谱。

### 验收标准

- 用户能在图谱页第一屏看到模式切换入口。
- 用户能在“全部”“通用”和任一已注册模式之间切换，页面不跳转。
- 切换模式后，图谱画布、图例、左侧筛选、右侧详情按当前模式同步更新。
- 新建知识时，用户能选择通用节点或任一已注册节点类型；不选择时保存为通用节点。
- 学术模式下，用户能按 domain 筛选学科知识，新学科出现后自动进入筛选列表。
- 没有注册模式时，图谱继续按默认能力可用，不出现空白页或死入口。

## 8. 调研队列 Research Queue

### 页面目标

调研队列回答“我让 Agent 调研的事现在到哪了，哪些报告能采纳进知识库”。Web 页面以查看和采纳为主，不承担对话外的新建调研入口。

### 第一次打开看到什么

页面顶部展示真实队列状态：

- 进行中数量
- 已完成数量
- 失败数量
- 已采纳报告数量
- 重点报告数量

下面是调研任务列表。每行显示：

- 调研主题
- 状态：进行中、已完成、失败、已归档
- 创建时间
- 完成时间
- 报告入口
- 重点标签
- 采纳状态
- 产出知识数

### 核心操作流：查看调研状态

1. 用户进入调研队列。
2. 默认看到未归档任务，按创建时间倒序。
3. 用状态筛选查看进行中、已完成、失败。
4. 点击完成任务，展开报告摘要和产出链接。
5. 点击报告链接打开报告或飞书文档。

### 核心操作流：采纳报告入库

1. 用户打开一个已完成任务。
2. 看报告摘要、来源、覆盖范围。
3. 点击“采纳入库”。
4. 系统提示：采纳后会拆解报告，生成领域知识条目。
5. 用户确认。
6. 任务状态显示“已采纳，正在入库”。
7. 入库完成后显示产出：知识条目数、Wiki 页面数、图谱节点数。
8. 用户点击“查看产出”进入领域知识库。

### “重点”标签

“重点”是质量标记，只说明报告信息密度高、覆盖面广。它不触发入库。

- Agent 可以自动标重点。
- 用户可以手动标重点或取消。
- 重点报告默认排在已完成列表前面。
- 筛选区支持只看重点。

### 信息层级

列表页优先显示状态和可行动作：

- 进行中：显示执行时间和当前阶段。
- 已完成未采纳：显示报告入口和采纳按钮。
- 已采纳：显示入库产物和跳转入口。
- 失败：显示失败原因和重试建议。
- 已归档：默认隐藏，筛选后可见。

### 关键交互说明

- 页面不放“手动创建调研任务”表单。用户通过对话发起调研，Web 负责查看、采纳、归档。
- 数据来源必须是真实任务记录和真实 reports/ 产物。
- 报告不存在时，明确显示“报告文件未找到”，不展示空链接。
- 采纳按钮只有已完成任务可用。
- 采纳后要能看到完整链路：调研报告 → 拆解 → 知识条目 → Wiki 页面 → 图谱。
- 归档只影响列表默认展示，不删除报告和知识。

## 9. 系统 System

### 页面目标

系统页回答“后台到底发生了什么，哪里出错了”。它面向低频排查，不干扰主知识工作流。

### 页面结构

系统页下放四个子入口：

- 操作日志
- 运行状态
- 导入记录
- 治理摘要

### 操作日志

操作日志记录真实事件：

- 知识条目创建、删除、合并。
- 文档导入事件：文件名、状态、产出知识数。
- 调研完成事件：主题、报告路径。
- 治理运行摘要：合并数量、清理数量、失败原因。
- 向量化批次完成。
- 图谱更新完成。

### 操作流

1. 用户进入系统。
2. 默认打开操作日志。
3. 按事件类型和日期筛选。
4. 点击一条日志，右侧展开详情。
5. 详情里提供跳转入口，例如进入某个文件、知识条目、调研报告。

### 关键交互说明

- 日志按日期分组，最新在上。
- 实时连接正常时显示“实时更新中”。
- 断线时显示“连接中断，正在重连”；重连失败后降级轮询。
- 系统页可以显示技术性错误，但要先翻译成人能懂的原因和动作。
- 操作日志不再放一级导航。

## 10. 设置 Settings

### 页面目标

设置页回答“哪些系统配置需要我管理”。它只放长期配置，不放日常工作流。

### 页面结构

设置页包含：

- 账号与密码：修改登录密码、会话管理。
- 系统词典：管理术语、别名、适用范围。
- 权限范围：查看知识域和角色访问范围。
- 导入导出：知识库 JSON 导出、导入、干跑预览。

### 核心操作流：系统词典

1. 用户进入设置 → 系统词典。
2. 查看术语列表。
3. 搜索术语或按 scope 筛选。
4. 新增术语时填写缩写、中文名、英文全称、定义、约束、别名、适用域。
5. 保存后术语进入知识体系，可参与注入。

### 核心操作流：导入导出

1. 用户进入设置 → 导入导出。
2. 选择导出范围：领域、类型、时间。
3. 下载 JSON。
4. 导入时先上传文件。
5. 系统先展示干跑报告：将新增多少、冲突多少、拒绝多少。
6. 用户确认后再真正导入。

### 关键交互说明

- 设置页不展示普通知识列表。
- 密码修改要说明登录会话影响。
- 导入前必须先展示校验结果，避免静默覆盖。
- 权限范围初期可以只读展示，不强行做复杂权限编辑。

## 11. 关键链路闭环

### 11.1 材料入库闭环

用户导入一批材料后，必须能看到：

文件上传成功 → 正在解析 → 正在提取 → 产出知识条目 → 生成 Wiki 页面 → 进入图谱 → 后续被搜索或注入命中

任何一步失败，都要在同一条链路上显示失败原因和恢复入口。

### 11.2 调研采纳闭环

用户通过对话发起调研后，必须能看到：

调研任务创建 → Agent 执行 → 报告完成 → 用户标记采纳 → 报告拆解 → 领域知识入库 → Wiki 页面生成 → 图谱更新

“采纳”是入库开关，“重点”只是质量标签。两者不能混用。

### 11.3 知识追溯闭环

任何知识条目都要能追溯：

这条知识来自哪个文件、哪一页、哪个时间戳、哪段报告或哪次对话；它现在属于哪个领域、哪个目录、哪些标签；它有没有被注入过、有没有进入图谱。

## 12. 需要修正的现有交互方向

当前 Web 已有不少能力，但信息架构需要收敛：

- 活动流从一级导航移到系统下的操作日志。
- 调研队列去掉 Web 新建调研表单，保留查看、采纳、归档、重点标记。
- 领域知识库要把“材料库”和“知识库”连成一条链，不能让用户在两个入口之间猜关系。
- 材料库列表改成紧凑表格，承载 20+ 文件批量处理。
- 仪表盘指标必须来自统一真实数据源，不能写死命中率和样例趋势。
- 一级导航不再按内部概念分组，直接按用户任务排列。
- 所有英文原始值在 UI 上翻译成中文。

## 13. Spec 覆盖说明

本设计覆盖以下产品能力：

- FR-B01 / FR-B02 / FR-B03 / FR-B05：知识存储、语义检索、关联、多维标签。
- FR-D01 / FR-D02 / FR-W07：调研任务生命周期、调研队列、采纳入库、重点标记。
- FR-E01 至 FR-E06：意图知识的提取、检索、注入、治理展示入口。
- FR-G01 至 FR-G05：图谱构建、洞察、可视化、图谱扩展注入的可见性。
- FR-W01 至 FR-W16：运行态概览、知识列表、详情、操作日志、文档导入、导航发现、图谱稳定性。
- FR-X01 至 FR-X04：权限范围、核心指标、导入导出。
