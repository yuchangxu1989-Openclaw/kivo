### 域 W-II：Web 调研管理（Research Management）

负责在 Web 驾驶舱中提供调研任务的查看、创建和管控能力，是引擎层域 D（自主调研）的 Web 操作界面。

#### FR-W03：调研队列查看与缺口报告

用户在 Web 端查看调研任务队列和缺口报告。

- AC1：调研任务列表展示任务状态（queued / running / completed / failed）、优先级、创建时间和预算消耗，支持按状态筛选和分页。
- AC2：缺口报告列表展示盲区主题、影响面（关联的未命中查询数）和填补进度，支持按影响面排序。
- AC3：缺口报告中的盲区条目提供"创建调研任务"入口，点击后跳转调研任务创建并预填充调研范围（与 FR-G03 AC1 联动）。

#### FR-W04：调研任务创建与管控

用户在 Web 端手动创建调研任务并管控执行。

- AC1：创建表单包含调研目标、范围、预期知识类型和资源预算，提交后任务进入调研队列。
- AC2：用户可取消 queued 或 running 状态的任务，取消后任务标记为 cancelled 并释放预算。
- AC3：用户可调整 queued 状态任务的优先级（上移/下移/置顶）。
- AC4：提供全局自动调研开关，关闭后调研队列进入静默模式（只入队不执行），开启后恢复正常调度。

### 域 W-III：Web 知识操作（Knowledge Operations）

负责在 Web 驾驶舱中提供知识条目的状态变更和内容编辑能力，补全从"浏览"到"操作"的闭环。

#### FR-W05：知识条目状态操作

用户在 Web 端对知识条目执行状态变更。

- AC1：支持将 active 条目标记为 deprecated（过时），将 pending 条目确认为 active 或拒绝为 archived。
- AC2：状态变更操作需要用户确认，确认后生成活动流事件。
- AC3：状态变更遵循知识条目状态机（§5.2），不允许非法状态流转；界面仅展示当前状态下可用的操作按钮。

#### FR-W06：知识条目内容编辑

用户在 Web 端编辑知识条目的摘要内容。

- AC1：编辑提交后生成新版本，旧版本保留在版本历史中。
- AC2：新版本必须经过 ConflictDetector 冲突检测，与新增知识走同一条检测管线；检测到冲突时阻止保存并提示用户。
- AC3：编辑操作使用乐观锁（expectedVersion），版本冲突时返回错误并提示用户刷新后重试。

### 域 X-I：访问控制与权限（Access Control）

落地 NFR-4.10~4.12 的访问控制要求，提供知识域级隔离和调用方鉴权能力。

#### FR-X01：知识域访问控制

按知识域隔离访问权限，防止敏感知识跨域泄露。

- AC1：每条知识条目归属一个知识域（domain），检索请求携带 callerRole，系统仅返回该角色有权访问的域内条目。
- AC2：域-角色映射通过配置文件静态定义（初期），支持后续升级为动态权限管理。
- AC3：规则分发遵循订阅范围约束，不向未订阅的 Agent 推送超出其域权限的规则内容。
- AC4：调研任务仅访问任务创建者有权限的信息源，不越权获取受限域的数据。

### 域 X-II：知识可观测性（Knowledge Observability）

为 §1 定义的 4 个成功指标提供度量采集机制，补全从"定义指标"到"采集数据"的链路。

#### FR-X02：核心指标度量采集

系统自动采集知识库运行指标，为仪表盘和健康评估提供数据源。

- AC1：采集知识检索命中率——记录每次检索请求的查询内容、返回条目数和是否命中（Top-K 中包含相关条目），支持按时间窗口聚合。
- AC2：采集缺口检测覆盖率——记录缺口检测运行结果（发现的盲区数、已填补数），支持与已知盲区基准对比。
- AC3：采集规则分发到达率——记录每次分发的目标订阅者数、成功送达数、失败数，支持按规则和时间窗口聚合。
- AC4：采集知识冲突解决率——记录冲突产生数、已解决数、待裁决数，确保未解决冲突可追踪。
- AC5：度量数据通过仪表盘聚合接口（FR-G01）对外暴露，不引入独立的度量存储。

### 域 X-III：通用知识导入导出（Knowledge Import/Export）

提供知识库级别的批量导入导出能力，覆盖全类型知识条目（非仅词典），支持跨实例迁移和备份恢复。

#### FR-X03：知识库批量导出

将知识库内容导出为可迁移的结构化文件。

- AC1：支持按知识域、知识类型、状态、时间范围筛选导出范围，导出格式为 JSON。
- AC2：导出内容包含知识条目完整数据（元数据、内容、关联关系、版本历史）和冲突记录。
- AC3：导出文件包含格式版本号，用于导入时的兼容性校验。

#### FR-X04：知识库批量导入

从导出文件中批量导入知识条目到目标知识库。

- AC1：导入前校验文件格式版本兼容性，不兼容时拒绝导入并提示原因。
- AC2：逐条导入时执行冲突检测（复用 ConflictDetector），冲突条目标记为 pending 待人工裁决，不静默覆盖。
- AC3：导入完成后生成导入报告，包含成功数、冲突数、跳过数、失败数及逐条明细。
- AC4：导入操作支持干跑模式（dry-run），仅输出报告不实际写入，用于预检。
