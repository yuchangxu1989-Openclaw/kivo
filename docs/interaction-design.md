# KIVO Web 交互设计

OpenClaw（ux-01 子Agent）2026-05-30

## 1. 页面 / 流程概览

### 1.1 全站工作台
- 目标用户：个人知识工作者、独立产品操盘者、Agent 开发者、知识治理用户。
- 使用时机：用户需要导入材料、管理知识、查找知识、配置知识注入、查看系统设置时进入。
- 页面目标：把知识生命周期中的录入、组织、检索、注入配置和基础设置串成稳定操作流。
- 成功结果：用户能看到已有知识资产，能上传或编辑内容，能搜索命中有效知识，能配置 Agent 注入范围，能完成密码、词典、导入导出等设置。
- 上游入口：登录后默认进入意图知识库；侧边栏、顶部搜索框、页面内跳转按钮也可进入对应页面。
- 下游去向：知识详情、Wiki 详情、材料详情、图谱聚焦视图、调研队列、操作日志。

### 1.2 核心页面范围
- 运行态概览：仪表盘展示知识总量、本周新增、类型分布、领域库摘要和趋势。
- 知识库管理：意图知识库、领域知识库、材料库、知识详情与 Wiki 详情。
- 知识图谱：图谱主页面、节点聚焦、关联浏览、搜索入口、模式切换。
- 调研管理：调研队列查看、筛选、报告链接、采纳、归档、重点标记；调研任务只从对话发起。
- 系统可观测：系统 / 操作日志列表、筛选、日期分组、事件详情。
- 登录认证：单密码登录、会话超时、主动退出后的返回登录。
- 知识注入配置：注入范围、预算、最低相关度、术语优先级、注入记录查看。
- 搜索 / 检索：全站知识搜索、列表内快速筛选、结果详情跳转。
- 设置页：密码修改、系统词典、权限范围、导入导出。

### 1.3 全站视觉与语言
- 视觉主调：白色底、黑色字，浅色边框，少量浅色状态标签。
- 禁止暗色主题、黑底、发光风格。
- 用户界面不出现开发者术语，不展示 embedding、worker、cron、provider、向量库参数。
- 状态文案必须讲人话：处理中、已完成、处理失败、需要重新上传、没有搜索结果。

## 2. 交互模式声明

### 2.1 全站模式
- 模式：混合式。
- 主输入方式：结构化表单、列表筛选、搜索框、文件上传。
- 辅助输入方式：顶部全站搜索、页面内快捷操作、确认弹窗。
- 采用原因：KIVO 管理的是结构化知识资产，创建、编辑、筛选、配置必须可控；搜索和调研入口允许自然语言表达。

### 2.2 页面模式分工
- 知识库管理：列表 + 详情 + 表单编辑。列表用于查找和筛选，详情用于阅读、溯源和操作。
- 知识注入配置：表单式配置 + 记录列表。配置必须结构化填写，记录用于解释注入发生过什么。
- 搜索 / 检索：搜索式。主输入是自然语言查询，筛选项用于缩小范围。
- 设置页：表单式。安全、词典、权限、导入导出都需要明确字段和确认动作。

### 2.3 对话与自然语言边界
- KIVO Web 不把知识管理做成纯聊天。
- 自然语言只用于搜索查询、调研描述、知识内容输入。
- 领域、类型、权限、预算、阈值、删除确认必须结构化操作。

## 3. 页面分区与信息结构

### 3.1 全站框架
- 左侧侧边栏：固定一级导航，顺序为仪表盘、意图知识库、领域知识库、知识搜索、知识图谱、材料库、调研队列、系统、设置。
- 顶部栏：全站搜索入口、退出登录。搜索按钮文案为「搜索知识...」。
- 主内容区：承载当前页面内容，白底黑字，列表和表单使用浅灰边框区分。
- 移动端：侧边栏折叠为菜单按钮，主操作保留在页面顶部。

### 3.2 知识库管理

#### 意图知识库
- 用途：管理从对话中沉淀的意图知识。
- 展示字段：标题、信号类型、来源、触发频率、创建时间、更新时间。
- 可执行动作：筛选、搜索、查看详情、编辑、删除。
- 联动关系：详情页展示正例、负例、相似句、触发记录；触发记录可跳到来源记录。

#### 领域知识库
- 用途：按 Subject 树管理领域空间、参考资料、Knowledge Entry 和 Wiki 页面。
- 展示字段：库名、描述、范围定义、条目数量、最近更新时间、加工链路统计。
- 页面结构：左侧目录树，右侧当前节点内容；顶部显示导入材料数、参考资料数、Knowledge Entry 数、图谱节点数、注入次数。
- 可执行动作：创建库、编辑库、删除库、创建 Subject、切换树形路径、切换标签筛选、查看条目、查看 Wiki 页面。
- 联动关系：选择 Subject 后，右侧同步展示该节点下的 Wiki 页面、参考资料、Knowledge Entry 和来源材料。

#### 材料库
- 用途：管理上传材料和处理结果。
- 展示字段：文件名、类型、大小、上传时间、处理状态、所属领域、产出统计。
- 可执行动作：上传材料、拖拽上传、筛选、排序、多选、重新处理、删除、查看原文、查看产出。
- 联动关系：处理完成后可跳到材料详情、领域知识库、Wiki 页面。

#### 知识 / Wiki 详情
- 用途：阅读知识内容、查看来源、查看关联、执行编辑和删除。
- 展示字段：标题、摘要、类型、来源数、知识点数、关系数、状态标记、正文、来源材料、关联知识、批注。
- 可执行动作：编辑知识点、删除、查看来源材料、在图谱中查看、添加批注、沉淀为知识。
- 联动关系：来源材料跳材料详情；关联知识跳对应详情；图谱入口聚焦当前节点。

### 3.3 知识注入配置
- 用途：控制 Agent 处理请求时注入哪些知识、注入多少、低相关内容是否过滤、术语是否优先。
- Spec 追踪：对应 FR-E01、FR-E05、FR-H02、FR-X01；页面只配置注入范围、预算、相关度和术语优先级，不新增独立业务域。
- 页面位置：设置页下的「知识注入配置」。如果当前实现先放在设置首页，入口文案保持这个名称。
- 页面分区：
  - 配置摘要区：显示当前状态、适用范围、最近更新时间。
  - 意图知识注入区：最低相关度、注入预算、优先级说明、启用状态。
  - 领域知识注入区：默认范围、图谱关联扩展、最大返回条数保护。
  - 术语注入区：术语优先、输出格式、预算占用说明。
  - 注入记录区：最近注入事件列表。
- 可执行动作：保存配置、恢复默认、查看记录、按目标 Agent 或时间筛选记录。
- 联动关系：配置保存后下一次 Agent 调用生效；记录可跳到被注入的知识条目。

### 3.4 搜索 / 检索
- 用途：用户用自然语言查找意图知识、领域知识、Wiki 页面和材料。
- 页面结构：顶部大搜索框，搜索历史，建议列表，结果 Tab，筛选栏，结果列表。
- 展示字段：标题、摘要片段、类型标签、所属领域、相关度指示、来源入口。
- 可执行动作：输入查询、选择建议、提交搜索、切换 Tab、筛选类型 / 领域 / 时间、点击结果。
- 联动关系：结果跳到意图详情、Wiki 详情、材料详情；无结果引导上传材料或提示用户回到对话中发起调研。

### 3.5 设置页
- 用途：管理安全、词典、权限范围、导入导出。
- 页面结构：设置首页以入口卡片展示；子页面使用标题区、说明区、操作区、结果区。
- 子入口：密码修改、系统词典、权限范围、导入导出。
- 可执行动作：修改密码、新增术语、编辑术语、删除术语、批量导入词典、导出词典、查看权限范围、导出知识库、导入前预览。
- 联动关系：词典更新后后续 Agent 注入使用新定义；导入导出结果进入操作日志。

## 4. 核心操作流

### 4.1 管理意图知识
1. 用户进入「意图知识库」。
2. 系统展示意图列表，按创建时间倒序排列；加载时显示骨架屏。
3. 用户输入筛选词或选择信号类型。
4. 系统立即缩小列表范围，筛选为空时显示「未找到匹配的意图知识」。
5. 用户点击某条意图。
6. 系统进入详情页，展示正文、正例、负例、相似句和触发记录。
7. 用户点击「编辑」。
8. 系统打开编辑表单，保存时按钮变为「保存中」。
9. 保存成功后回到详情阅读态，顶部提示「意图知识已更新」。
10. 保存失败时保留用户输入，提示失败原因，提供「重试」。

### 4.2 管理领域知识库
1. 用户进入「领域知识库」。
2. 系统展示左侧目录树和右侧默认节点内容。
3. 空库时展示「还没有领域知识，上传材料后系统会自动归类到这里」，提供「上传材料」。
4. 用户点击「新建领域知识库」。
5. 系统打开表单，字段为名称、描述、知识范围定义。
6. 名称为空或重名时，提交按钮禁用或展示字段错误。
7. 创建成功后，新库出现在列表中并自动选中。
8. 用户在目录树中创建或选择 Subject。
9. 系统刷新右侧内容，展示当前节点下的参考资料、Knowledge Entry、Wiki 页面和来源材料。
10. 用户删除知识库时，系统弹确认框，明确显示该库下知识条目数量，并说明条目会变为未分类。

### 4.3 上传和处理材料
1. 用户进入「材料库」。
2. 用户点击「上传材料」或拖拽文件到页面。
3. 系统展示上传区域，提示支持 PDF、图片、文本 / Markdown、音频、视频，单文件上限 50 MB。
4. 用户选择目标领域，默认值为「自动识别」。
5. 文件选择后立即开始上传，材料卡片插入列表顶部，状态为「上传中」。
6. 上传成功后进入处理流程，卡片展示阶段：已登记 → 识别材料 → 提取内容 → 形成参考资料 → 萃取知识 → 归入领域 → 生成 Wiki → 建立关系。
7. 长材料展示「已处理 N / M 个片段」。
8. 处理完成后展示参考资料 N 条、Knowledge Entry N 条、Wiki 页面 N 个、关系 N 条。
9. 用户点击「查看产出的内容」「进入领域知识库」「打开 Wiki 页面」继续浏览。
10. 上传或处理失败时，卡片展示失败原因和「重试」按钮；旧成功产出不被失败状态覆盖。

### 4.4 配置知识注入
1. 用户进入「设置」并打开「知识注入配置」。
2. 系统加载当前配置；加载中显示「正在读取注入配置」。
3. 用户调整意图知识最低相关度、注入预算、领域范围、术语优先开关。
4. 用户点击「保存配置」。
5. 系统立即禁用保存按钮，显示「保存中」。
6. 保存成功后提示「配置已保存，将在下一次 Agent 处理请求时生效」。
7. 保存失败时提示原因，配置表单停留在编辑态，保留用户改动。
8. 用户点击「恢复默认」。
9. 系统弹确认框，确认后恢复默认值但不立即保存；用户仍需点击「保存配置」。
10. 用户打开「注入记录」。
11. 系统展示最近记录：时间、目标 Agent、场景摘要、注入条目数、来源类型。

### 4.5 搜索知识
1. 用户点击侧边栏「知识搜索」或顶部「搜索知识...」。
2. 系统进入搜索页，搜索框 placeholder 为「搜索知识...」，下方显示最近 10 条搜索历史。
3. 用户输入查询。
4. 系统展示基于已有知识标题的建议。
5. 用户按回车或点击「搜索」。
6. 系统进入搜索中状态，保留查询词和筛选条件。
7. 成功后按 Tab 展示「全部」「领域知识」「意图知识」「材料」。
8. 每条结果展示标题、摘要片段、类型标签、所属领域、相关度指示。
9. 用户点击结果跳到对应详情页。
10. 无结果时展示「未找到相关知识」，提供「上传材料」和「在对话中发起调研」说明；KIVO Web 不提供新建调研任务表单。
11. 检索能力不可用时展示「知识搜索暂时不可用，请检查检索服务配置」，不展示空结果冒充成功。

### 4.6 管理设置
1. 用户进入「设置」。
2. 系统展示设置入口卡片：密码修改、系统词典、权限范围、导入导出、知识注入配置。
3. 用户进入「系统词典」。
4. 系统展示术语列表，支持搜索和 scope 筛选。
5. 用户点击「新增术语」。
6. 表单展开，字段为术语、定义、别名、scope。
7. 保存成功后提示「术语已新增」，列表刷新。
8. 删除术语时弹确认框，确认后删除。
9. 用户进入「导入导出」。
10. 导出时先选择范围；导入时先上传文件并预览校验结果，不兼容时拒绝导入并展示原因。

## 5. 动作定义

### 动作：搜索知识
- 位置：顶部栏、知识搜索页主搜索框。
- 触发条件：输入框非空。
- 禁用条件：查询为空、正在搜索。
- 点击后即时反馈：按钮显示「搜索中」，结果区显示骨架屏。
- 成功结果：展示结果 Tab 和结果列表，记录进入最近搜索历史。
- 失败结果：展示错误提示和「重试」。
- 是否可撤销：不可撤销，但可重新搜索。

### 动作：上传材料
- 位置：材料库顶部按钮、空状态主按钮、拖拽上传区。
- 触发条件：选择或拖入支持格式文件。
- 禁用条件：文件超过 50 MB、格式不支持、上传队列已满。
- 点击后即时反馈：材料卡片立即出现在列表顶部，显示「上传中」。
- 成功结果：进入材料处理流程。
- 失败结果：卡片展示失败原因和「重试」。
- 是否可撤销：上传中可取消；处理开始后只能删除材料。

### 动作：重新处理
- 位置：材料卡片、材料详情页。
- 触发条件：材料已完成或处理失败。
- 禁用条件：材料正在上传或处理中。
- 点击后即时反馈：弹确认框；确认后状态变为「处理中」。
- 成功结果：新产出替换旧产出，旧产出保留到新产出成功生成。
- 失败结果：显示失败原因，旧成功产出继续可见。
- 是否可撤销：处理开始后不可撤销。

### 动作：删除材料
- 位置：材料卡片更多菜单、材料详情页。
- 触发条件：材料存在。
- 禁用条件：删除请求进行中。
- 点击后即时反馈：确认框显示「将同时删除该材料产出的 N 条知识和 N 个 Wiki 页面」。
- 成功结果：材料从列表移除，相关产物按后端规则清理。
- 失败结果：保留材料，提示原因。
- 是否可撤销：不可撤销。

### 动作：新建领域知识库
- 位置：领域知识库页面顶部。
- 触发条件：用户点击按钮。
- 禁用条件：无。
- 点击后即时反馈：打开表单弹窗。
- 成功结果：新库出现在领域列表中并自动选中。
- 失败结果：字段级提示重名、缺字段或保存失败。
- 是否可撤销：可通过删除知识库移除。

### 动作：删除领域知识库
- 位置：领域知识库详情操作区。
- 触发条件：选中用户创建的知识库。
- 禁用条件：系统内置顶层分类不可删除。
- 点击后即时反馈：确认框显示该库下条目数量和影响说明。
- 成功结果：知识库移除，条目变为未分类。
- 失败结果：保留原状，提示失败原因。
- 是否可撤销：不可撤销。

### 动作：编辑知识点
- 位置：知识详情页、领域知识详情页、材料产出列表。
- 触发条件：知识条目存在。
- 禁用条件：当前版本已被其他操作更新且未刷新。
- 点击后即时反馈：进入编辑态，展示标题、正文、标签、来源等字段。
- 成功结果：生成新版本，页面展示新内容。
- 失败结果：版本冲突时提示「内容已被更新，请刷新后重试」。
- 是否可撤销：可通过版本历史还原。

### 动作：沉淀为知识
- 位置：参考资料、Wiki 页面、调研报告内容区域。
- 触发条件：用户选中一段内容或点击条目操作。
- 禁用条件：内容为空、当前对象处理中。
- 点击后即时反馈：显示「正在萃取知识候选」。
- 成功结果：通过门禁后生成 Knowledge Entry；未通过时保留为参考资料并提示原因。
- 失败结果：展示失败原因和「重试」。
- 是否可撤销：生成后的 Knowledge Entry 可删除。

### 动作：保存注入配置
- 位置：设置 / 知识注入配置。
- 触发条件：配置表单有改动且字段合法。
- 禁用条件：保存中、字段缺失、数值越界。
- 点击后即时反馈：按钮显示「保存中」，表单暂时禁用。
- 成功结果：提示「配置已保存，将在下一次 Agent 处理请求时生效」。
- 失败结果：恢复可编辑，保留改动，展示失败原因。
- 是否可撤销：可恢复默认后重新保存。

### 动作：新增术语
- 位置：设置 / 系统词典。
- 触发条件：点击「新增术语」。
- 禁用条件：术语或定义为空。
- 点击后即时反馈：展开表单或提交保存。
- 成功结果：术语出现在词典列表，提示「术语已新增」。
- 失败结果：保留输入，展示失败原因。
- 是否可撤销：可删除术语。

### 动作：修改密码
- 位置：设置 / 密码修改。
- 触发条件：填写当前密码、新密码、确认新密码。
- 禁用条件：字段为空、两次新密码不一致、保存中。
- 点击后即时反馈：按钮显示「保存中」。
- 成功结果：提示「密码已修改，请使用新密码登录」。
- 失败结果：提示当前密码错误或保存失败。
- 是否可撤销：不可撤销，只能再次修改。

## 6. 字段与内容展示规则

### 6.1 全站导航字段
- 导航名称：固定展示中文名称，不使用英文缩写。
- 当前页面：侧边栏高亮当前导航项。
- 顶部搜索：placeholder 固定为「搜索知识...」。
- 退出登录：按钮文案「退出」。

### 6.2 意图知识字段
- 标题：一句话概括，超过 20 字截断并显示省略号，hover 展示完整标题。
- 信号类型：纠偏 / 强调 / 声明 / 规则 / 偏好，必显。
- 来源：来源对话引用，缺失时显示「来源未记录」。
- 触发频率：展示「命中 N 次」，无记录显示「命中 0 次」。
- 正例 / 负例：列表展示；空值分别显示「暂无正例」「暂无负例」。
- 相似句：标签样式展示，过多时折叠为「展开全部」。

### 6.3 领域知识库字段
- 库名：必显，全局唯一；超长单行截断。
- 描述：最多展示两行，超长折叠。
- 知识范围定义：详情中完整展示，列表中摘要展示。
- 条目数量：展示数字，空库显示 0。
- 最近更新时间：统一格式为 YYYY-MM-DD HH:mm。
- 加工链路统计：展示导入材料数、参考资料数、Knowledge Entry 数、图谱节点数、注入次数。
- Subject 名称：树节点展示名称和下属条目数。

### 6.4 材料字段
- 文件名：必显，保留扩展名，超长中间截断。
- 文件类型：展示 PDF、图片、文本、音频、视频，不展示 MIME 原始值。
- 文件大小：小于 1 MB 显示 KB，大于等于 1 MB 显示 MB。
- 上传时间：YYYY-MM-DD HH:mm。
- 处理状态：上传中 / 处理中 / 已完成 / 处理失败。
- 所属领域：未识别时显示「未分类」。
- 产出统计：完成后展示参考资料 N 条、Knowledge Entry N 条、Wiki 页面 N 个、关系 N 条。
- 失败原因：展示用户可读原因，不能只显示错误码。

### 6.5 知识 / Wiki 详情字段
- 标题：≤20 字展示；详情页标题可 hover 看完整标题。
- 摘要：1-3 句话，空值显示「暂无摘要」。
- 正文：保留段落结构。
- 类型标签：fact、methodology、decision、experience、intent、meta 展示为中文标签。
- 来源材料：文件名、页码 / 时间戳 / 区域定位；无来源显示「暂无来源记录」。
- 关联知识：按关系类型分组展示，关系来源显示为系统推断、同材料共现、用户确认。
- AI 状态标记：AI 生成 / 用户已编辑 / 存在争议。

### 6.6 注入配置字段
- 启用状态：开启 / 关闭，必显。
- 最低相关度：用百分比或滑块展示，不展示底层评分算法。
- 注入预算：展示为「最多注入约 N 字内容」或「预算 N」，避免解释内部 token 细节；开发实现可映射到 token。
- 目标范围：全部知识、仅意图知识、仅领域知识、指定领域。
- 术语优先：开启 / 关闭，说明「预算不足时优先保留术语定义」。
- 图谱关联扩展：开启 / 关闭，说明「同时带出强相关知识」。
- 注入记录：时间、目标 Agent、场景摘要、注入条目数、来源类型、结果状态。

### 6.7 搜索结果字段
- 标题：关键词高亮；无标题时显示内容前 20 字。
- 摘要片段：命中内容高亮，最长两行。
- 类型标签：领域知识、意图知识、材料、Wiki 页面。
- 所属领域：无领域显示「未分类」。
- 相关度指示：高 / 中，不展示具体分数。
- 时间：创建或更新时间。
- 跳转目标：领域知识到 Wiki 或知识详情，意图知识到意图详情，材料到材料详情。

### 6.8 设置字段
- 术语：必填，列表中单行展示。
- 定义：必填，列表中最多两行。
- 别名：逗号分隔输入，标签展示。
- scope：空值保存为「全局」。
- 权限范围：以只读列表展示角色、可访问领域、说明。
- 导出范围：知识域、知识类型、时间范围。
- 导入预览：文件名、格式版本、可导入条目数、冲突或不兼容原因。

## 7. 页面状态机

### 7.1 初始态
- 触发条件：用户第一次进入页面，尚未发起加载或输入。
- 页面展示：页面标题、说明、主要操作按钮；搜索页展示搜索框和历史记录；空库页面展示引导。
- 可做动作：导航、搜索、上传、新建、进入设置子页。

### 7.2 加载态
- 触发条件：页面首次拉取数据、切换筛选、打开详情、读取配置。
- 页面展示：骨架屏或「加载中…」，保留页面标题和导航。
- 可做动作：可切换导航；当前数据相关按钮禁用。

### 7.3 空态
- 触发条件：接口成功返回空列表或筛选后无结果。
- 页面展示：
  - 意图知识库：「系统会从你和 Agent 的对话中自动提取意图知识，目前还没有」。
  - 领域知识库：「还没有领域知识，上传材料后系统会自动归类到这里」。
  - 材料库：「上传文档、图片、音频或视频，系统自动解析资料并萃取高价值知识」。
  - 搜索页：「未找到相关知识」。
- 可做动作：上传材料、新建知识、清空筛选；需要调研时，页面提示用户回到对话中发起，不在 Web 内创建调研任务。

### 7.4 错误态
- 触发条件：接口返回错误、权限不足、检索服务不可用、保存失败。
- 页面展示：用户可读错误标题、原因摘要、重试按钮。
- 可做动作：重试、返回上一页、修改输入、进入设置检查配置。

### 7.5 处理中
- 触发条件：上传、材料处理、知识萃取、搜索、保存配置、导入预览等异步动作进行中。
- 页面展示：
  - 材料处理显示阶段进度和片段进度。
  - 搜索显示骨架结果。
  - 保存显示按钮 loading。
  - 导入预览显示校验中。
- 可做动作：允许离开页面；会破坏当前提交的重复按钮禁用。

### 7.6 已完成
- 触发条件：异步动作成功完成。
- 页面展示：
  - 材料卡片展示产出统计和三个跳转入口。
  - 搜索展示结果列表。
  - 保存配置展示成功提示。
  - 导入预览展示可导入结果。
- 可做动作：查看详情、继续搜索、重新处理、编辑、删除、跳转图谱。

### 7.7 超时 / 中断
- 触发条件：长任务超过预期无更新、网络中断、浏览器刷新后任务状态未同步。
- 页面展示：
  - 材料卡片显示「处理时间较长，系统仍在继续」。
  - 若确认失败，显示「处理失败」和失败原因。
  - 搜索超时显示「搜索暂时没有返回结果」。
- 可做动作：刷新状态、重试、返回列表、查看操作日志。

### 7.8 部分完成
- 触发条件：材料部分片段已处理，或批量操作部分成功。
- 页面展示：已完成数量、总数量、失败数量；逐项展示失败原因。
- 可做动作：重试失败项、查看已成功结果、返回列表。

## 8. AI 特有反馈规则

### 8.1 AI 输出内容
- 材料识别、内容提取、参考资料形成、Knowledge Entry 萃取、Wiki 页面、关联关系、意图知识、相似句、搜索相关度、注入候选均可能由 AI 或语义检索流程生成。
- 系统确定值包括文件名、文件大小、上传时间、用户手动输入、删除确认、登录状态、权限范围。

### 8.2 AI 标识方式
- Wiki 页面卡片和详情必须展示状态标记：AI 生成 / 用户已编辑 / 存在争议。
- 关联知识展示来源类型：系统推断 / 同材料共现 / 用户确认。
- 搜索结果展示相关度指示「高 / 中」，不展示底层分数。
- 注入记录展示来源路径：语义命中 / 图谱关联扩展 / 术语优先。

### 8.3 输入来源说明
- 材料产出要能回到文件、页码、时间戳或区域定位。
- 意图知识要能回到来源对话引用。
- Wiki 页面要展示贡献该页面的来源材料。
- 注入记录要展示本次注入基于哪次请求场景和哪些知识条目。

### 8.4 不确定性提示
- 低置信归类展示「未分类」或「建议归类」，由用户手动调整。
- 存在争议的 Wiki 页面展示状态标记，不混入确定结论。
- 自动推算的重要性只显示为建议，不覆盖用户手动设置。
- 检索无结果时不能说知识不存在，只提示「未找到相关知识」。

### 8.5 重试、修正、继续生成
- 材料处理失败可「重试」。
- 参考资料、Wiki 页面、调研报告可「沉淀为知识」。
- 知识内容可「编辑知识点」。
- 版本冲突时提示刷新后重试。
- 搜索可以修改查询或筛选后再次提交。

### 8.6 覆盖 / 保留策略
- 重新处理材料时，旧产出保留到新产出成功生成。
- 编辑知识点生成新版本，历史版本可还原。
- 系统治理自动处理冲突，但用户可在版本历史中看到合并、覆盖记录。
- 保存注入配置只影响下一次 Agent 调用，不改写历史注入记录。

## 9. 长任务与异步可见性

### 9.1 任务阶段列表
- 材料导入：已登记 → 识别材料 → 提取内容 → 形成参考资料 → 萃取知识 → 归入领域 → 生成 Wiki → 建立关系。
- 调研采纳入库：已采纳 → 拆解报告 → 形成参考资料 → 生成 Wiki → 萃取 Knowledge Entry 候选 → 去重 / 合并 → 入库完成。
- 导入导出：读取文件 → 校验格式 → 预览结果 → 执行导入 / 导出 → 完成。
- 知识萃取：收到标记 → 萃取候选 → 门禁判断 → 写入或保留为参考资料。

### 9.2 列表页显示
- 材料库列表卡片必须展示当前阶段、失败原因、产出统计。
- 调研队列展示状态：进行中 / 已完成 / 已归档。
- 操作日志记录材料导入事件、调研完成事件、治理运行摘要、向量化批次完成。
- 搜索页搜索中状态不清空旧查询和筛选。

### 9.3 详情页显示
- 材料详情页显示文件摘要、处理进度、产出统计、内容列表。
- Wiki 详情页显示来源材料、关联知识、批注和 mini 图谱。
- 知识详情页显示来源引用、版本历史、关联关系。
- 注入配置页显示最近注入记录。

### 9.4 是否可离开页面
- 材料处理、调研采纳、导入导出、知识萃取都允许离开页面。
- 离开页面不取消后台任务。
- 回到列表页后，系统根据任务状态继续展示进度或终态。

### 9.5 返回后恢复方式
- 页面刷新后重新读取材料状态、调研状态、导入记录和操作日志。
- 处理中材料继续显示阶段进度。
- 已完成材料显示产出统计和跳转入口。
- 失败材料显示失败原因和重试入口。

### 9.6 完成通知
- 页面内使用轻提示：材料处理完成、配置已保存、术语已新增、导入完成。
- 长任务完成后在操作日志新增事件。
- 当前页面仍打开时，列表项在 3 秒内刷新为最新状态。

### 9.7 失败恢复入口
- 材料失败：卡片和详情页提供「重试」。
- 搜索失败：结果区提供「重试」。
- 保存失败：按钮恢复可用，表单保留输入。
- 导入失败：预览页展示不兼容原因，不执行导入。
- 注入配置读取失败：提供「重试」和「返回设置」。

## 10. Agent 产品专项规则

### 10.1 生成过程可见
- 请求已收到：上传材料后立即出现材料卡片；沉淀为知识后立即出现「正在萃取知识候选」提示。
- 阶段性进度：材料处理必须展示 8 个阶段；长材料展示已处理片段数 / 总片段数。
- 流式输出：KIVO Web 的知识管理页面不要求流式正文输出；阶段进度必须持续更新。
- 中途取消：上传中可取消；处理开始后不提供取消，只提供删除材料或等待完成。
- 超时提示：材料处理长时间无更新时显示「处理时间较长，系统仍在继续」，并提供「刷新状态」。

### 10.2 不确定性可见
- AI 生成内容必须打标，用户编辑后状态改为「用户已编辑」。
- 关联关系必须显示来源类型，系统推断关系允许用户确认或删除。
- 搜索结果只显示相关度高 / 中，不包装成确定答案。
- 自动归类低置信时归入「未分类」，用户可手动调整。
- 用户可通过编辑、重新处理、沉淀为知识、删除关系来修正结果。

### 10.3 异步任务可恢复
- 后台继续跑时，列表页展示进行中状态，不让任务消失。
- 刷新页面后，从后端状态恢复任务阶段和统计。
- 从列表进入详情时，能看到当前阶段、历史产出、失败原因。
- 完成和失败都写入操作日志。
- 失败支持重试；旧成功产出在新处理完成前保持可见。

### 10.4 混合交互模式分工
- 必须结构化填写：知识库名称、描述、范围定义、Subject、知识类型、筛选条件、权限范围、注入预算、最低相关度、术语字段、导入导出范围。
- 允许自然语言输入：搜索查询、知识正文、Wiki 正文编辑、调研描述、术语定义、批注。
- 表单提交后不进入对话修正；修正通过编辑表单完成。
- 搜索查询不反写结构化配置；用户选择筛选项后才更新 URL 或页面状态。

## 11. 对接假设

### 11.1 核心实体
- Knowledge Entry：知识条目，字段至少包含 id、type、title、content、status、domain、source、createdAt、updatedAt。
- Intent Entry：意图知识，字段至少包含 id、title、signalType、content、positives、negatives、similarSentences、source、hitCount、createdAt。
- Domain Space：领域知识库，字段至少包含 id、name、description、scopeDefinition、entryCount、updatedAt。
- Subject：领域目录节点，字段至少包含 id、name、parentId、level、entryCount。
- Material：材料，字段至少包含 id、fileName、fileSize、fileType、status、stage、processedChunks、totalChunks、domain、outputs、errorMessage。
- Wiki Page：Wiki 编译页面，字段至少包含 id、title、summary、status、sourceCount、entryCount、relationCount、updatedAt。
- Injection Config：注入配置，字段至少包含 enabled、scope、budget、minRelevance、termPriority、graphExpansion、updatedAt。
- Injection Record：注入记录，字段至少包含 id、time、targetAgent、sceneSummary、entryCount、sourceType、status。

### 11.2 状态字段
- 所有知识展示视图默认只展示 active 条目。
- 材料状态：uploading、processing、done、failed。
- 材料阶段使用人话阶段枚举，不直接展示后端内部阶段名。
- Wiki 状态：AI 生成、用户已编辑、存在争议。
- 搜索状态：idle、loading、success、empty、error。
- 保存状态：idle、saving、success、error。

### 11.3 刷新策略
- 材料处理中：列表页 3-5 秒刷新一次或使用 SSE 推送，完成后停止。
- 搜索结果：用户主动提交后刷新，不自动轮询。
- 注入记录：进入页面读取，用户点击「刷新记录」手动更新。
- 操作日志：支持实时推送，断线重连后补发断线期间事件。


## 12. 开发验收清单

- 全站为白色底 + 黑色字，侧边栏和卡片只使用浅色背景。
- 仪表盘展示知识总量、本周新增、类型分布、领域库数量、趋势和领域摘要卡，统计数必须与子页面真实数据一致。
- 知识图谱页面支持缩放、平移、节点拖拽、节点聚焦、预览卡片、模式切换、筛选、返回全局视图和失败重试。
- 调研队列只展示通过对话发起的真实调研任务，不提供 Web 内新建调研任务入口；已完成任务支持查看报告、采纳、归档和重点标记。
- 系统 / 操作日志按事件类型筛选、按日期分组、支持详情查看、实时更新和断线补发。
- 登录页为单密码登录，无用户名；覆盖密码为空、密码错误、登录中、会话超时和主动退出状态。
- 侧边栏一级导航顺序与 spec 一致。
- 顶部搜索入口在所有工作台页面可见，文案为「搜索知识...」。
- 意图知识库与领域知识库页面结构、URL 前缀、卡片字段独立，不混排。
- 材料上传后卡片立即出现在列表顶部。
- 材料处理展示 8 个用户可读阶段，不展示内部技术名。
- 处理完成材料展示产出统计和三个跳转入口。
- 搜索页展示最近 10 条搜索历史、建议、Tab、筛选和结果字段。
- 搜索无结果时展示上传材料入口和「在对话中发起调研」说明，不提供 Web 内新建调研任务表单。
- 检索能力不可用时展示错误，不用空结果冒充成功。
- 知识详情页不展示置信度字段。
- 列表超长字段截断并可查看完整内容。
- 删除材料、删除知识库、删除术语都有确认框。
- 编辑知识点保存失败时保留用户输入。
- 知识注入配置保存后提示下一次 Agent 调用生效。
- 注入记录能看到时间、目标 Agent、场景摘要、注入条目数、来源类型。
- 系统词典支持搜索、scope 筛选、新增、编辑、删除、批量导入、批量导出。
- 导入导出先预览或选择范围，失败时展示原因。
- 所有列表都有加载态、空态、错误态、已完成态。
- 长任务允许离开页面，回来后能恢复状态。
- AI 生成内容和系统确定值有清楚区分。
- 用户界面不出现 embedding、worker、cron、provider、FTS、BGE 等内部技术词。

## 13. 一级页面交互定义

### 13.1 仪表盘

#### 页面定位
- 页面名称：仪表盘。
- URL 建议：`/dashboard`。
- Spec 追踪：FR-W01、FR-W12。
- 使用时机：用户进入工作台后快速确认知识库当前规模、增长趋势和各领域知识库状态。
- 页面目标：用真实统计展示知识资产运行态，并提供到核心子页面的跳转。
- 成功结果：用户能看到知识总量、本周新增、知识类型分布、领域知识库摘要，点击任一卡片可进入对应明细页。

#### 布局与导航
- 顶部标题区：标题「仪表盘」，副标题「查看知识库当前运行状态」。右侧放「刷新」按钮。
- 指标摘要区：四张横向指标卡，分别为「知识总量」「本周新增」「知识类型分布」「领域知识库」。
- 趋势区：展示本周与上周的变化。趋势只展示「较上周增加 N」「较上周减少 N」「与上周持平」，不展示内部统计口径说明。
- 知识类型分布区：展示所有已定义知识类型的数量，必须包含 Wiki 类型，不遗漏任何 entry type。
- 领域知识库摘要区：以卡片列表展示领域知识库，字段为库名、条目数、最近更新时间。
- 快捷入口区：展示「意图知识库」「领域知识库」「知识搜索」「知识图谱」「材料库」入口，点击进入对应页面。
- 视觉规则：白色页面背景，黑色文字，卡片用浅灰边框；趋势上升、下降只用轻量标签，不使用暗色底。

#### 操作流
1. 用户点击侧边栏「仪表盘」。
2. 系统展示页面骨架屏，并读取统计数据。
3. 数据返回后，系统展示四张指标卡、趋势区、类型分布和领域库摘要。
4. 用户点击「知识总量」卡片，进入知识搜索页，默认展示全部知识。
5. 用户点击「本周新增」卡片，进入知识搜索页，默认筛选本周新增。
6. 用户点击某个知识类型，进入知识搜索页，默认筛选该类型。
7. 用户点击领域知识库摘要卡片，进入对应领域知识库详情页。
8. 用户点击「刷新」，按钮显示「刷新中」，完成后更新页面数据并提示「数据已更新」。
9. 刷新失败时，保留上一次成功数据，顶部提示「仪表盘数据更新失败，请重试」。

#### 动作定义
- 动作：刷新仪表盘。
  - 位置：标题区右侧。
  - 触发条件：用户点击「刷新」。
  - 禁用条件：正在刷新。
  - 即时反馈：按钮文案变为「刷新中」。
  - 成功结果：统计和摘要卡片更新，提示「数据已更新」。
  - 失败结果：保留旧数据，展示「仪表盘数据更新失败，请重试」。
  - 是否可撤销：不可撤销。
- 动作：进入明细。
  - 位置：指标卡、类型分布项、领域库摘要卡。
  - 触发条件：卡片有可跳转目标。
  - 禁用条件：数据加载中或目标缺失。
  - 即时反馈：卡片出现轻量按下态。
  - 成功结果：进入对应子页面并带上筛选条件。
  - 失败结果：提示「页面暂时无法打开」。
  - 是否可撤销：可通过浏览器返回。

#### 状态反馈
- 初始态：显示标题、副标题和骨架屏。
- 加载态：四张指标卡显示骨架；导航和顶部搜索保持可用。
- 空态：所有统计为 0 时，展示「还没有知识资产，上传材料或从对话中沉淀知识后，这里会自动更新」，提供「上传材料」入口。
- 错误态：展示「仪表盘暂时无法加载」，提供「重试」。
- 已完成：展示真实统计；统计数必须与对应子页面一致。
- 中断态：如果刷新中网络断开，保留旧数据并提示「网络已断开，恢复后可重试」。

#### 文案和字段规则
- 知识总量：来自 entries 实际记录总数，展示整数；空值显示 0。
- 本周新增：展示本周新增条目数，格式「N 条」。
- 知识类型分布：展示类型中文名和数量；类型数量为 0 也要显示。
- 领域知识库数量：展示库数量，格式「N 个」。
- 趋势：只展示与上周相比的数量变化，格式「较上周增加 N」「较上周减少 N」「与上周持平」。
- 领域摘要卡：库名超长单行截断，条目数显示「N 条」，最近更新时间统一 YYYY-MM-DD HH:mm。
- 禁止文案：不出现缓存、entries 表、内部任务名、统计 SQL、技术实现名。

### 13.2 知识图谱主页面

#### 页面定位
- 页面名称：知识图谱。
- URL 建议：`/graph`。
- Spec 追踪：FR-G03、FR-G06、FR-W16、FR-P03、FR-W12。
- 使用时机：用户想浏览知识之间的关联、查看某条知识的一跳关系、从图谱发现孤立节点和桥接节点。
- 页面目标：提供可缩放、可平移、可聚焦、可筛选的图谱浏览界面。
- 成功结果：用户能搜索并定位节点，查看关联关系，跳到知识详情，也能一键返回全局视图。

#### 布局与导航
- 顶部工具栏：页面标题「知识图谱」、图谱搜索框、模式切换、筛选按钮、「返回全局视图」按钮。
- 图谱画布区：占主内容区大部分空间，展示节点和边；白色背景，节点和边使用浅色区分，不使用黑底画布。
- 左侧筛选抽屉：知识域、知识类型、时间范围、关系类型；筛选后图谱就地刷新。
- 右侧预览卡片：点击节点后出现，展示节点标题、类型、所属领域、摘要、关联数量、来源状态和「查看详情」。
- 底部状态条：展示当前视图节点数、边数、最后更新时间；加载时显示「正在加载图谱」。
- 洞察标记：孤立节点、桥接节点、稀疏社区用浅色标签或描边标记，必须有图例说明。

#### 节点展示与关联浏览
- 节点按知识类型区分颜色或图标，颜色必须保持浅色可读。
- 边按关系类型区分线型或浅色标签。
- 鼠标滚轮或触控板缩放图谱。
- 拖拽空白区域平移图谱。
- 拖拽节点只改变当前视图位置，不改业务数据。
- 点击节点后，高亮该节点和直接关联节点，其余节点降低视觉权重。
- 双击节点进入对应详情页；点击边时，右侧预览卡片展示关系类型、关联强度、来源类型；不能编辑关系，除非知识详情页已有对应编辑能力。

#### 搜索入口
- 搜索框 placeholder：「搜索图谱节点...」。
- 输入时展示节点标题建议，建议项字段为标题、类型、所属领域。建议来自已有图谱节点标题。
- 提交搜索后，画布自动定位到最匹配节点并打开右侧预览卡片。
- 搜索无结果时展示「未找到匹配节点」，提供「清空搜索」。

#### 模式切换
- 模式切换入口常驻顶部工具栏，选项为「全部」「通用」和已注册模式。
- 首次进入时默认「全部」。
- 切换模式不跳页，图谱就地重新渲染。
- 如果没有注册模式，仅展示「全部」「通用」；不显示空下拉。
- 模式数据加载失败时，保留切换前视图并提示「图谱视图切换失败，请重试」。

#### 操作流
1. 用户点击侧边栏「知识图谱」。
2. 系统展示工具栏和图谱骨架，底部状态为「正在加载图谱」。
3. 图谱加载完成后，系统展示全局视图。
4. 用户输入节点关键词并提交搜索。
5. 系统定位到节点，打开右侧预览卡片，高亮一跳关联。
6. 用户点击「查看详情」。
7. 系统进入对应知识详情页。
8. 用户从详情页返回图谱时，保留最近一次筛选和聚焦状态。
9. 用户点击「返回全局视图」。
10. 系统清除节点聚焦，恢复全局图谱。
11. 用户连续点击多个节点时，系统取消旧请求或忽略旧响应，只展示最后一次点击结果。
12. 图谱加载失败时，系统展示错误提示和「重试」。

#### 动作定义
- 动作：搜索节点。
  - 位置：顶部工具栏搜索框。
  - 触发条件：输入非空。
  - 禁用条件：图谱初始加载中。
  - 即时反馈：搜索框显示「搜索中」。
  - 成功结果：定位节点并打开预览卡片。
  - 失败结果：展示「搜索失败，请重试」。
  - 是否可撤销：可清空搜索或返回全局视图。
- 动作：点击节点。
  - 位置：图谱画布节点。
  - 触发条件：节点可见。
  - 禁用条件：同一节点详情加载中。
  - 即时反馈：节点出现选中描边，预览卡显示骨架。
  - 成功结果：展示节点预览和一跳关联。
  - 失败结果：展示「节点详情加载失败」，提供「重试」。
  - 是否可撤销：点击空白处或返回全局视图。
- 动作：返回全局视图。
  - 位置：顶部工具栏。
  - 触发条件：当前存在聚焦节点、搜索条件或局部子图。
  - 禁用条件：全局视图已展示。
  - 即时反馈：按钮按下态。
  - 成功结果：清除聚焦，恢复全局节点和边。
  - 失败结果：展示「全局视图加载失败，请重试」。
  - 是否可撤销：可重新搜索或点击节点。
- 动作：切换模式。
  - 位置：顶部工具栏。
  - 触发条件：用户选择模式。
  - 禁用条件：模式列表加载中。
  - 即时反馈：显示「正在切换视图」。
  - 成功结果：就地渲染对应子图。
  - 失败结果：保留原视图并提示失败。
  - 是否可撤销：可切回原模式。

#### 状态反馈
- 初始态：展示工具栏、筛选入口和图谱骨架。
- 加载态：画布展示节点骨架，底部状态显示「正在加载图谱」。
- 空态：展示「还没有可展示的知识关系」，提供「上传材料」和「进入领域知识库」。
- 错误态：展示「图谱加载失败」，提供「重试」。
- 聚焦态：被选节点和一跳节点高亮，右侧预览卡可见。
- 切换中：模式切换、筛选或节点展开时禁用冲突操作。
- 超时态：展示「图谱加载时间较长，请稍后重试」，保留重试入口。
- 多次重试失败：提示「仍然无法加载图谱，请刷新页面后再试」，并记录操作日志。

#### 文案和字段规则
- 节点标题：超长截断，hover 展示完整标题。
- 节点类型：展示中文类型标签。
- 所属领域：无值显示「未分类」。
- 关联数量：格式「N 个关联」。
- 关系类型：展示中文关系名；未知关系显示「其他关联」。
- 来源类型：系统推断、同材料共现、用户确认。
- 洞察标签：孤立节点、桥接节点、稀疏社区。
- 预览摘要：最多三行，空值显示「暂无摘要」。
- 禁止文案：不展示图布局算法、请求参数、内部字段名、底层评分。

### 13.3 调研队列页面

#### 页面定位
- 页面名称：调研队列。
- URL 建议：`/research`。
- Spec 追踪：FR-D01、FR-D02、FR-W07、FR-P05、FR-W12。
- 使用时机：用户想查看通过对话发起过的调研任务，打开报告，对已完成报告执行采纳、归档或重点标记。
- 页面目标：展示真实调研任务生命周期，保证用户能看见任务状态和产出链接。
- 边界：调研任务只通过对话发起。页面无创建入口、无编辑入口、无调研配置入口。

#### 布局与导航
- 顶部标题区：标题「调研队列」，副标题「查看通过对话发起的调研任务」。
- 触发说明区：浅色提示条，文案「要发起新调研，请在对话中说明调研主题。这里负责展示任务状态和产出」。提示条只说明入口，不提供创建表单。
- 筛选区：状态筛选「全部」「进行中」「已完成」「失败」「已归档」；重点筛选「全部」「重点」。
- 列表区：任务列表按创建时间倒序展示。
- 分页区：展示当前页、总页数、总任务数。
- 详情抽屉：点击任务行后打开，展示任务描述、状态时间线、产出链接、失败原因、采纳记录。

#### 触发方式
- 用户在对话中发起调研后，系统自动创建调研任务，进入队列。
- 任务进行中时，页面只显示状态和创建时间。
- 任务完成后，页面展示产出链接。
- Web 页面不能把搜索无结果、图谱洞察或按钮点击直接变成新建调研任务；这些入口只能提示「请在对话中发起」。图谱洞察相关提示仅作为 FR-G02 的后续承接说明，不在本页面新增调研创建动作。

#### 操作流
1. 用户点击侧边栏「调研队列」。
2. 系统展示筛选区和列表骨架。
3. 数据返回后，系统展示任务描述、状态、创建时间、产出链接。
4. 用户按状态筛选。
5. 系统刷新列表，筛选为空时展示「没有符合条件的调研任务」。
6. 用户点击已完成任务的产出链接。
7. 系统打开报告文件或文档链接。
8. 用户点击「采纳」。
9. 系统弹确认框，文案「采纳后，系统会把这份报告送入 Wiki 准入流程」。
10. 用户确认后，任务进入采纳处理中，按钮显示「采纳中」。
11. 准入流程启动成功后，状态提示「已采纳，正在进入 Wiki 准入流程」。
12. 用户点击「归档」。
13. 系统将任务从默认列表隐藏，可通过「已归档」筛选查看。
14. 用户点击「标为重点」或「取消重点」。
15. 系统更新重点标签；重点标签不触发入库。

#### 动作定义
- 动作：筛选调研任务。
  - 位置：筛选区。
  - 触发条件：用户选择筛选项。
  - 禁用条件：列表初始加载中。
  - 即时反馈：列表显示骨架。
  - 成功结果：展示筛选后的任务列表。
  - 失败结果：提示「调研任务加载失败，请重试」。
  - 是否可撤销：可切回「全部」。
- 动作：打开产出。
  - 位置：已完成任务行和详情抽屉。
  - 触发条件：任务状态为已完成且产出链接存在。
  - 禁用条件：任务未完成、产出链接缺失。
  - 即时反馈：链接按下态。
  - 成功结果：打开报告。
  - 失败结果：提示「报告暂时无法打开」。
  - 是否可撤销：可返回队列。
- 动作：采纳报告。
  - 位置：已完成任务行、详情抽屉。
  - 触发条件：任务已完成、未采纳、产出链接存在。
  - 禁用条件：任务进行中、失败、已归档且未恢复、采纳处理中。
  - 即时反馈：确认框；确认后按钮显示「采纳中」。
  - 成功结果：触发 Wiki 准入流程，任务标记为已采纳。
  - 失败结果：保留未采纳状态，展示失败原因和「重试」。
  - 是否可撤销：采纳触发后不可撤销，后续入库结果由准入流程控制。
- 动作：归档任务。
  - 位置：任务行更多菜单、详情抽屉。
  - 触发条件：任务已完成或失败。
  - 禁用条件：任务进行中、归档请求中。
  - 即时反馈：确认框。
  - 成功结果：任务从默认列表隐藏。
  - 失败结果：任务保留原状态，提示原因。
  - 是否可撤销：可在已归档筛选中恢复显示。
- 动作：标为重点。
  - 位置：任务行、详情抽屉。
  - 触发条件：任务已完成。
  - 禁用条件：请求中。
  - 即时反馈：标签切换为「重点」。
  - 成功结果：任务保留重点标签。
  - 失败结果：恢复原标签，提示原因。
  - 是否可撤销：可取消重点。

#### 状态反馈
- 初始态：展示标题、触发说明和筛选区。
- 加载态：列表骨架；筛选暂时禁用。
- 空态：无任何任务时展示「还没有调研任务。你可以在对话中发起调研，完成后会出现在这里」。
- 筛选空态：展示「没有符合条件的调研任务」，提供「清空筛选」。
- 进行中：状态标签「进行中」，展示创建时间，不展示产出链接。
- 已完成：状态标签「已完成」，展示产出链接和可用操作。
- 失败：状态标签「失败」，展示失败原因；不提供采纳。
- 已归档：状态标签「已归档」，默认列表隐藏。
- 采纳处理中：展示「正在进入 Wiki 准入流程」。
- 错误态：展示「调研队列加载失败」，提供「重试」。

#### 文案和字段规则
- 任务描述：必显，最多两行，超长折叠。
- 状态：进行中、已完成、失败、已归档；如果底层返回其他状态，展示「未知状态」并记录日志。
- 创建时间：YYYY-MM-DD HH:mm。
- 产出链接：已完成且存在时显示「查看报告」；缺失时显示「报告链接未记录」。
- 重点标记：标签文案「重点」，只表示质量参考，不触发入库。
- 采纳状态：未采纳、采纳中、已采纳、采纳失败。
- 分页：展示「第 N / M 页，共 X 条」。
- 禁止文案：不写「新建调研」「创建任务」「配置调研 Agent」这类 Web 创建入口。

### 13.4 系统 / 操作日志页面

#### 页面定位
- 页面名称：系统 / 操作日志。
- URL 建议：`/system/logs`。
- Spec 追踪：FR-W04、FR-W12、FR-Z08。
- 使用时机：用户需要回看知识条目变动、文档导入、调研完成、治理运行摘要和批次完成记录。
- 页面目标：提供低频、客观、可筛选的系统行为记录。
- 成功结果：用户能按事件类型和日期找到日志，打开详情并查看关键字段。

#### 布局与导航
- 一级导航显示「系统」。进入系统后默认展示「操作日志」。
- 顶部标题区：标题「操作日志」，副标题「查看知识库关键动作记录」。
- 筛选区：事件类型、日期范围、关键词搜索、结果状态。
- 列表区：按日期分组展示日志，每组标题为日期。
- 详情抽屉：点击日志行后打开，展示事件摘要、时间、对象、结果、失败原因、关联入口。
- 实时提示区：有新事件到达时，在列表顶部显示「有 N 条新日志」，用户点击后插入列表。

#### 操作流
1. 用户点击侧边栏「系统」。
2. 系统进入操作日志页面并展示筛选区、列表骨架。
3. 数据返回后，系统按日期分组展示日志。
4. 用户选择事件类型筛选。
5. 系统刷新列表，筛选条件保留在页面上。
6. 用户点击某条日志。
7. 系统打开详情抽屉，展示事件字段和关联入口。
8. 新事件产生时，页面顶部显示「有 N 条新日志」。
9. 用户点击提示条。
10. 系统把新日志插入对应日期分组。
11. 客户端断线重连后，系统自动补发断线期间事件，并显示「已补充断线期间的日志」。
12. 加载失败时，系统展示错误提示和「重试」。

#### 动作定义
- 动作：筛选日志。
  - 位置：筛选区。
  - 触发条件：选择事件类型、日期或输入关键词。
  - 禁用条件：初始加载中。
  - 即时反馈：列表显示加载态。
  - 成功结果：展示符合条件的日志。
  - 失败结果：提示「日志筛选失败，请重试」。
  - 是否可撤销：可清空筛选。
- 动作：查看日志详情。
  - 位置：日志列表行。
  - 触发条件：日志存在。
  - 禁用条件：详情加载中。
  - 即时反馈：右侧抽屉打开并显示骨架。
  - 成功结果：展示事件详情。
  - 失败结果：提示「日志详情加载失败」。
  - 是否可撤销：可关闭抽屉。
- 动作：查看新日志。
  - 位置：列表顶部实时提示条。
  - 触发条件：有新事件到达。
  - 禁用条件：无新事件。
  - 即时反馈：提示条消失，列表刷新。
  - 成功结果：新日志插入列表。
  - 失败结果：提示「新日志加载失败，请刷新」。
  - 是否可撤销：不可撤销。

#### 状态反馈
- 初始态：展示标题、筛选区和列表骨架。
- 加载态：按日期分组区域展示骨架行。
- 空态：展示「还没有操作日志」。
- 筛选空态：展示「没有符合条件的日志」，提供「清空筛选」。
- 实时连接中断：展示浅色提示「实时更新已断开，正在重连」。
- 重连成功：展示「已恢复实时更新」。
- 补发成功：展示「已补充断线期间的日志」。
- 错误态：展示「操作日志加载失败」，提供「重试」。

#### 文案和字段规则
- 事件类型：知识条目变动、文档导入、调研完成、治理运行摘要、向量化批次完成、认证事件。
- 时间：统一 YYYY-MM-DD HH:mm:ss。
- 事件摘要：一句话描述发生了什么；不展示内部任务名。
- 对象：展示知识标题、文件名、调研主题或批次摘要。
- 结果状态：成功、失败、部分完成。
- 失败原因：必须是用户可读文案；没有失败时不展示。
- 关联入口：可跳到知识详情、材料详情、调研报告或设置页；目标不存在时显示「关联对象不可用」。
- 日期分组：今天、昨天、YYYY-MM-DD。
- 禁止文案：不展示内部技术词、数据库表名、服务进程名、堆栈信息。

### 13.5 登录页

#### 页面定位
- 页面名称：登录页。
- URL 建议：`/login`。
- Spec 追踪：FR-W11、FR-Z08。
- 使用时机：用户未登录、会话超时、主动退出后重新进入 KIVO Web。
- 页面目标：用单密码完成认证，不要求用户名。
- 成功结果：密码正确后进入工作台；首次 onboarding 完成后默认进入意图知识库。

#### 布局与导航
- 页面背景：纯白色。
- 登录卡片：居中展示，浅灰边框，黑色文字。
- 标题：KIVO。
- 副标题：「输入访问密码进入知识工作台」。
- 输入区：一个密码输入框，placeholder「请输入密码」。
- 操作区：主按钮「登录」。
- 辅助区：错误提示、会话超时提示、加载状态提示。
- 不展示用户名输入框、注册入口、第三方登录入口。

#### 认证流程
1. 未登录用户访问任意受保护页面。
2. 系统跳转到登录页，并保留原访问目标。
3. 用户输入密码。
4. 密码为空时，「登录」按钮禁用。
5. 用户点击「登录」。
6. 按钮显示「登录中」，输入框暂时禁用。
7. 密码正确时，系统建立会话并跳转到原访问目标；没有原访问目标时进入意图知识库。
8. 密码错误时，系统留在登录页，提示「密码不正确，请重新输入」。
9. 会话超时后，用户操作受保护页面时跳回登录页，提示「登录已过期，请重新登录」。
10. 用户主动退出后，系统清除会话并回到登录页，提示「已退出登录」。

#### 动作定义
- 动作：提交密码。
  - 位置：登录卡片主按钮。
  - 触发条件：密码输入框非空。
  - 禁用条件：密码为空、正在提交。
  - 即时反馈：按钮文案变为「登录中」，输入框禁用。
  - 成功结果：进入工作台或原访问目标。
  - 失败结果：展示错误提示，输入框恢复可编辑。
  - 是否可撤销：提交中不可撤销，失败后可重新输入。
- 动作：退出登录。
  - 位置：工作台顶部栏。
  - 触发条件：用户已登录。
  - 禁用条件：退出请求中。
  - 即时反馈：按钮显示「退出中」。
  - 成功结果：回到登录页，展示「已退出登录」。
  - 失败结果：提示「退出失败，请重试」。
  - 是否可撤销：退出成功后需重新登录。

#### 状态反馈
- 初始态：展示密码输入框和「登录」按钮。
- 输入态：用户输入非空密码后，按钮可点击。
- 密码为空：按钮禁用，不弹错误。
- 提交中：按钮显示「登录中」。
- 登录成功：跳转，不额外弹窗。
- 密码错误：输入框下方展示「密码不正确，请重新输入」。
- 会话超时：卡片顶部展示「登录已过期，请重新登录」。
- 退出后：卡片顶部展示「已退出登录」。
- 网络错误：展示「暂时无法登录，请检查网络后重试」。

#### 文案和字段规则
- 密码字段：必填，输入内容隐藏显示。
- 主按钮：固定文案「登录」。
- 错误文案：密码错误、登录过期、网络失败三类分开展示。
- 会话跳转：登录成功后优先返回原访问目标；无目标时进入意图知识库。
- 对比度：主按钮和文字满足 WCAG AA 对比度要求。
- 禁止文案：不出现用户名、注册、OAuth、token、session id、cookie、认证中间件等开发者词。
