# 产品需求规格 - KIVO 走查 P2 + P3 清理

OpenClaw(Claude Code 子Agent)/ 2026-05-24

## 1. 背景与目标

KIVO 陌生小白用户走查(`reports/kivo-stranger-walkthrough-2026-05-24.md`)暴露 P2/P3 共 5 项细节问题,集中表现为「数字与列表口径不一致」「文案承诺未实现的入口」「空内容卡片让用户怀疑系统坏了」「设置入口要靠猜 URL」「空目录树误导用户必须手工整理」。这些问题不阻断核心流程,但拉低首次体验质量,必须在面向陌生用户开放前清理掉。

本规格只覆盖这 5 项小修复,不引入新功能、不改动数据模型,全部在 web 层(仪表盘 API、wiki 列表 API、TopNav、SpaceManager、目录树空状态文案)完成。

## 2. 范围

- 仪表盘领域知识库数字与左侧学科树长度对齐。
- 主导航入口与用户菜单文案严格对应。
- wiki 列表卡片只展示有内容的知识页面。
- Header 用户菜单 dropdown 暴露明显的「设置」入口。
- 空目录树补齐引导文案。

不在范围:搜索相关性、上传状态反馈、图谱渲染、登录跳转等 P0/P1 项;走查 P2-2「领域 wiki 内部搜索结果标关键词匹配」(已被独立修复任务覆盖)。

## 3. 功能需求

### FR-1:领域知识库统计与学科树口径对齐
- 现象:仪表盘「领域知识库 1」,左侧学科树却列出 5 个学科节点(含未关联任何 entry/material 的占位)。
- 修复:仪表盘 `/api/v1/dashboard/summary` 已基于 `wiki_spaces` 计数,无需变更;学科树 `/api/subjects` 输出端只保留「有 entries 承载」的节点,即 `entries.subject_id = id` 至少有 1 条记录的节点;同时保留通向这些叶子的祖先链路。其余节点(如未抽取出 entry 的占位学科)一律从树里隐藏。
- AC1:左侧学科树长度 = 仪表盘领域知识库数 + 各领域下真实承载学科节点数;当前 DB 状态下,左侧只能看到「概率论与数理统计 6」「高等数学 2」两个叶子。
- AC2:当一个父节点的所有后代都被过滤掉时,父节点本身也被过滤;仅保留通向「有承载」叶子的链路。

### FR-2:主导航与用户菜单文案口径收口
- 现象:主导航实际渲染 5 个 Tab(原始资料库、领域 wiki、意图知识库、知识图谱、知识搜索),但 TopNav 里有一段隐藏文案 `KIVO 二级入口 / 调研、用户理解、系统词典、操作日志、配置、学科域管理`,让陌生用户以为还有 8 个入口。
- 修复:用户菜单 dropdown 改为只暴露当前已实装且陌生用户能用的入口(设置、用户理解、调研、退出登录),删除 TopNav 中那段误导性 sr-only 文案。
- AC1:TopNav 不再渲染「KIVO 二级入口」字样;用户菜单条目数 ≤ 5。
- AC2:用户菜单中每个入口点击后能进入对应页面,不出现 404。

### FR-3：wiki 列表过滤空内容卡片
- 现象：wiki 列表显示「暂未提取出合格知识条目」的空壳卡片（pipeline-worker 在切片为 0 时填入的占位 content，几十字 + `**提取知识条目**: 0` 标记）。
- 修复：`/api/wiki/spaces/[id]/entries` 列表查询返回结果中过滤掉「空壳 wiki_page」：`content` 为空、`content` 不足 30 字符、或 `content` 包含 `**提取知识条目**: 0` 标记的页面，一律不出现在列表里。
- AC1：列表 `meta.total` 与渲染卡片数都不再包含上述空壳页面。
- AC2：当前 DB 下列表卡片数从 13 减少到 10（`认知科学.pdf` / `蛋白质结构预测...pdf` / `高斯课堂：二重积分练习...pdf` 这三个占位页面被过滤），其余卡片照常显示。

### FR-4:用户菜单暴露设置入口
- 现象:陌生用户找不到 `/settings`,只能猜 URL。
- 修复:Header 的用户菜单 dropdown 顶部固定列出「设置」链接,链接到 `/settings`。
- AC1:从任意 dashboard 页面打开用户菜单 → 点击「设置」 → 跳转到 `/settings` 且 HTTP 200。
- AC2:「设置」条目带 `Settings` 图标,与其它菜单项视觉一致。

### FR-5:空目录树引导文案
- 现象:领域 wiki 选中某个 space 后,若目录树为空,页面只显示「新建目录」按钮,陌生用户以为必须手工整理。
- 修复:DirectoryManager 在 `tree.children` 为空(不含目录子节点)时,渲染一段引导文案「上传材料后会自动建分类,无需手工整理」;「新建目录」按钮保留但不强调(弱化为次要操作样式)。
- AC1:当 space 下没有任何 wiki_directory 节点时,目录树区域显示该引导文案。
- AC2:「新建目录」按钮仍可点击进入新建对话框,仅视觉权重降低。

## 4. 非功能需求

- 视觉:白底黑字,沿用现有 slate 调色板。
- 兼容:不破坏现有 API 契约;FR-3 仅在响应层过滤;FR-1 在 `/api/subjects` 输出层过滤,不写库。
- 数据:禁止在任何 placeholder/seed 中写入「概率论」「数学」等具体学科名,避免再次污染。
- 验证:`npm run build` 通过;`curl /kivo/dashboard`、`/kivo/settings`、`/kivo/wiki` 返回 200。
