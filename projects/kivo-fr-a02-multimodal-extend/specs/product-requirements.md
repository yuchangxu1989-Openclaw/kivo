# KIVO FR-A02 多模态扩展 - 产品需求规格

OpenClaw（sa-01 子Agent）/ 2026-05-24

---

## 1. 用户人群

KIVO 的全部用户。重点画像：

- 主要使用图片、音频、视频积累一手资料的用户：拍课件、拍板书、拍手写笔记、录课堂、录讲座、看录课视频。
- 同时混用文档与多模态材料的用户：一本教材搭一段录音，一份讲义搭一张拍的板书图，需要把两路输入收进同一个知识库。
- 已经把素材丢进 KIVO，但只看到"ready / pending"、得不到结构化笔记的用户。

---

## 2. 痛点

- 上传图片只能看到原图缩略图，公式、定义、表格不会被还原成可检索的文本知识。
- 录音、录课、录屏视频丢进来后没有自动转写入库，相当于黑盒文件。
- 板书、手写笔记、网课截屏混在一起时，系统不知道该走哪条提取链路，最后大量素材停在 pending。
- 同一个素材里既有讲解音轨又有公式板书，目前要么只取音、要么只取图，知识被切成两半，关联不上原始内容。
- 素材入库后无法回到"哪一秒、哪一帧、哪一页"做溯源，用户不敢信赖提取出的条目。

---

## 3. 原始需求

在不改变 FR-A02 已有验收标准的前提下，把多模态输入打通成一条可执行的提取主链路：图片走 OCR、音频走本地 Whisper、视频走"音轨 + 关键帧"双通道，最终统一转成文本片段进入知识抽取流水线，并保留对原始媒体的位置溯源（页/坐标/时间戳/帧号）。

本 spec 只负责"上传 → 路由 → 解析 → 文本片段交给后续提取管线"这一段，不重写知识入库、向量化、质量门禁等下游能力。

---

## 4. 用户体验流

1. 用户从飞书或 Web 端上传一份素材（图片 / 音频 / 视频 / 含公式扫描页）。
2. 系统在落盘时先识别文件类型，把素材路由到对应的解析通道，状态从 ready 进入解析中。
3. 图片素材自动 OCR 识别，公式还原为 LaTeX，和原图位置一起记录。
4. 音频素材自动转写，按时间窗口切成文本片段，每段保留时间戳。
5. 视频素材双通道处理：音轨抽出转写、关键帧抽出做 OCR，两路结果合并成同一份带时间戳的文本流。
6. 解析完成后，所有产物以"文本片段 + 媒体定位元数据"的形式进入已有的知识抽取流水线。
7. 用户在素材详情页可以看到解析进度、产出文本片段数、跳回原始图/原始音频时间点/原始视频帧的链接。
8. 解析失败的素材进入失败列表，用户可一键重试，并能看到失败原因摘要。

---

## 5. 功能需求

### FR-A 图片 OCR 路由

针对图片类素材（拍照、截图、扫描页、含公式图片）执行 OCR 与公式还原。

- AC1：本地 OCR 引擎为默认通道，处理常规印刷体、手写笔记、板书、流程图截图等场景，OCR 结果与原图坐标一起写入解析产物。
- AC2：含公式的图片走"先识别公式区域 → 公式还原为 LaTeX → 非公式部分走常规 OCR"的组合，公式块和文字块在产物里分别可寻址。
- AC3：用户可在配置中启用付费增强 OCR 通道作为备选，对本地 OCR 失败或低置信度区域自动二次识别，启用与否对其他逻辑无副作用。
- AC4：图片解析产物保留原图引用位置（页号或图片 ID + 区域坐标），下游知识条目可一路回链到具体区域。
- AC5：解析失败时素材状态明确标记为失败并附原因（图片不可读 / 内容空白 / 引擎不可用），不静默挂死在解析中。

### FR-B 音频转写

针对音频类素材执行本地语音转写，输出带时间戳的文本片段。

- AC1：默认使用本地轻量级 Whisper 模型完成转写，无需外网与额外密钥，配置项可调整语言与模型档位。
- AC2：转写产物按合理时间窗口（如句子或语义停顿）切成文本片段，每个片段附 start/end 时间戳。
- AC3：单段音频时长上限与单文件大小上限沿用 FR-A02 已有约束，超限素材直接进入失败列表并给出可读原因。
- AC4：转写产物作为统一文本片段进入知识抽取流水线，下游不需要为音频专门再写一份提取逻辑。
- AC5：转写失败的素材保留原始音频可下载，方便用户人工核对或换通道重试。

### FR-C 视频处理

针对视频类素材执行音轨转写与关键帧 OCR 双通道处理，并合并为统一时间线。

- AC1：上传后自动抽取音轨，复用 FR-B 的转写能力，得到带时间戳的文本片段。
- AC2：按场景切换或固定步长抽取关键帧图片，复用 FR-A 的 OCR 能力，得到带帧号/时间戳的文本片段。
- AC3：双通道结果按统一时间轴合并，相邻同义内容（音轨说出 + 板书写出同一个公式）会被标记为同一知识来源，避免重复入库。
- AC4：视频解析产物中每条文本片段都能回链到对应时间点，用户从知识条目一跳能定位到原视频片段。
- AC5：解析失败时按通道分别记录失败原因（音轨失败 / 关键帧失败 / 整体失败），允许只重试失败通道。

### FR-D 上传路由器

按 mime 类型与文件后缀决定素材进入哪条解析通道，是多模态扩展的总入口。

- AC1：所有上传入口（飞书、Web、URL 抓取后转入）共用同一个路由器，路由结果作为素材落盘元数据持久化。
- AC2：路由优先级为 mime 类型为主、文件后缀为辅，遇到二者冲突时以 mime 为准并记录冲突日志。
- AC3：识别到不支持的类型时，素材状态直接标记为不支持，给用户可读提示，不进入解析中状态。
- AC4：路由结果包含通道名（图片 / 音频 / 视频 / 文档 / 不支持）与解析参数（如视频抽帧步长），下游解析器只读取该结构，不再自己嗅探类型。
- AC5：同一素材重新上传时路由结果稳定，不会因二次上传被分到另一条通道，保证溯源关系不漂移。
